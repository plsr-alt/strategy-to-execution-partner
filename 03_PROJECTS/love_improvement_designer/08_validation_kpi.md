## 改善レバー（売上が出たら必ず回す）

### レバー1：タイトル改善（CTR）
- noteのタイトルを2週間ごとにA/Bテスト
- Xの投稿文言で反応を見てからnoteタイトルに反映
- 数字・疑問形・ネガティブ訴求のバリエーション

### レバー2：導線改善
- Xプロフィール → 固定ポスト → note → 診断 → DM の流れを最適化
- 各ステップの離脱ポイントを特定
- 固定ポスト・告知文を週1で改善

### レバー3：内容改善（満足度）
- note購入者の反応を確認（スキ数、コメント）
- 「読者が今日動ける」粒度になっているか再チェック
- 実例・ケーススタディの追加

### レバー4：商品ラインナップ拡充
- 1本目で反応が出たテーマの深掘り版
- テーマ別分冊（メッセージ編、デート編、プロフ編）
- まとめマガジン化（3本→1マガジン）

## 撤退基準

| 期間 | 条件 | 判定 |
|------|------|------|
| 2週間 | X投稿への反応（いいね+RT）= 0 | **撤退** |
| 3週間 | note PV 累計100未満 | **撤退** |
| 1ヶ月 | note売上 = 0円 | **テーマ変更** |

### 撤退時のアクション
1. 数値データを `99_ARCHIVE/` に退避
2. 学びを3行で記録（何が刺さらなかったか）
3. 次のテーマ候補を2つ出す
4. 翌日から次のDay1開始

> **撤退 ≠ 失敗。学習の完了。次の弾へ。**
