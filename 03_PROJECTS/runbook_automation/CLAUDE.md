# PJ: 手順書キャプチャ自動化 — CLAUDE.md

## プロジェクト概要
キャプチャ画像から項目を自動認識し、値を抽出して入力・反映する仕組みの検討〜社内提案。

## ゴール
1. 技術調査の完了と方式比較表の作成
2. MVP推奨案とロードマップの策定
3. 社内提出用資料（10枚）の完成

## ディレクトリ構造
```
runbook_automation/
├── CLAUDE.md              ← このファイル（PJルール）
├── research/              ← 調査ログ
├── decision_log/          ← 意思決定ログ
├── experiment_log/        ← 実験・検証ログ
├── docs/                  ← 運用ドキュメント・変更履歴
└── proposal/              ← 社内提出用資料
```

## 作業ルール
- 不明点は仮置き（[仮]マーク）で進め、後で確認
- すべての判断は decision_log/ に記録
- 実験結果は experiment_log/ に記録
- ユーザーは記録作業をしない。Claudeが自動で記録する
