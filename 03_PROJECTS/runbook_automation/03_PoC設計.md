# 手順書自動化 — PoC設計
_作成: 2026-02-20_

---

## 1. PoCの目的

「技術的に実現可能か」ではなく、**「業務に使える精度と速度で動くか」** を検証する。

---

## 2. 成功条件（定量）

| 指標 | 成功ライン | 計測方法 |
|------|-----------|---------|
| **抽出精度（フィールド単位正解率）** | ≥ 85% | テストセット50件の手動正解と比較 |
| **処理時間（画像投入〜Sheets書込）** | ≤ 60秒/件 | Lambdaログのbegin-endタイム |
| **ヒューマンレビュー発生率** | ≤ 30%（信頼度0.85未満の割合） | Sheetsのフラグ集計 |
| **手動入力削減率（時間ベース）** | ≥ 50% | 従来作業時間 vs. レビューのみ時間 |
| **エラー率（Lambdaクラッシュ等）** | ≤ 5% | CloudWatchエラーログ |

---

## 3. 検証項目

### Week 1: インフラ & 基本パイプライン
- [ ] S3バケット・Lambda・TextractのIAM権限設定
- [ ] 画像投入 → Textract呼び出し → JSON取得 の疎通確認
- [ ] Amazon BedrockのAPI接続確認（Claude 3.5 Haiku）
- [ ] Google Sheets APIの認証・書き込み確認

### Week 2: 抽出精度の検証
- [ ] 対象手順書（1種類）のフィールド定義書を作成
- [ ] Textract出力 + LLMプロンプトで正規化JSON生成
- [ ] テストセット20件で精度測定（第1回）
- [ ] 低精度フィールドの原因特定（OCR品質 / LLMプロンプト / フォーマット）

### Week 3: 精度改善 & レビューフロー
- [ ] 前処理（画像補正）の追加効果を計測
- [ ] プロンプトチューニング（few-shot例の追加）
- [ ] 信頼度スコアの閾値調整（0.85 が適切か）
- [ ] Sheetsのレビューフロー動作確認（フラグ・色分け）
- [ ] テストセット30件（追加）で精度測定（第2回）

### Week 4: 統合テスト & 評価
- [ ] エンドツーエンドで50件処理
- [ ] 処理時間・エラー率の計測
- [ ] 担当者による実業務ウォークスルー（UXフィードバック）
- [ ] 成功条件の達成確認
- [ ] PoC結果レポート作成

---

## 4. 想定スケジュール（4週間）

```
Week 1  [インフラ構築]
  ├─ Day 1-2: AWS環境構築（S3/Lambda/Textract/Bedrock）
  ├─ Day 3:   Sheets API設定
  └─ Day 4-5: パイプライン疎通テスト

Week 2  [精度検証 Round 1]
  ├─ Day 1:   フィールド定義書作成（業務担当者と合意）
  ├─ Day 2-3: プロンプト設計・実装
  └─ Day 4-5: テストセット20件の精度測定・分析

Week 3  [改善 & レビューフロー]
  ├─ Day 1-2: 前処理・プロンプト改善
  ├─ Day 3:   信頼度閾値調整
  └─ Day 4-5: 追加30件テスト・Sheetsフロー確認

Week 4  [統合テスト & まとめ]
  ├─ Day 1-2: エンドツーエンド50件処理
  ├─ Day 3:   担当者ウォークスルー
  └─ Day 4-5: 結果レポート・次フェーズ提案
```

## 5. 必要リソース

### 人員
| 役割 | 工数 | 担当 |
|------|------|------|
| エンジニア（パイプライン実装） | 1名 × 4週間 | 社内 or 外注 |
| 業務担当者（フィールド定義・テスト） | 0.2名 × 4週間 | 社内（業務側） |
| プロジェクトオーナー（判断・承認） | スポット | 事業責任者 |

### AWS費用（PoC期間4週間の概算）
| サービス | 推定費用 |
|---------|---------|
| AWS Textract（~500ページ） | 〜 $7 |
| Amazon Bedrock Claude Haiku（~500リクエスト）| 〜 $2 |
| AWS Lambda | 無料枠内 |
| S3（数GB） | 〜 $1 |
| **合計** | **〜 $10〜30（PoC期間）** |

### その他
- Google Workspace アカウント（Sheets API用）
- テスト用手順書キャプチャ画像 50件以上（業務担当者から提供）
