**作成日**: 2026-02-20
**ステータス**: 調査・計画フェーズ
**担当テックリード**: Claude（参謀）

---

## 1. プロジェクト背景

| 項目 | 現状 |
|------|------|
| 対象業務 | 社内手順書に従った入力・登録作業 |
| 作業形態 | 画面キャプチャを見ながら人手で入力 |
| 主な問題 | 属人化 / 入力ミス / 作業時間増大 |
| 手順書フォーマット | 画像（画面キャプチャ）含む文書 |

---

## 2. ゴール

> **キャプチャ画像から項目を自動認識し、値を抽出・入力できる仕組みを構築する**

### 短期（MVP）
- 入力支援：抽出結果をオペレーターに提示し、確認後に入力
- 入力ミス削減・作業時間20%以上削減

### 中期
- 半自動：高信頼度の項目は自動入力、低信頼度のみ人手確認

### 長期
- 全自動：例外のみ人手介入。フィードバックループで精度継続改善

---

## 3. スコープ

### IN（MVP対象）
- 対象手順書：社内の特定業務ジャンル1〜2種（仮定：経費申請・勤怠登録）
- 入力項目：固定フォーム形式（ラベル＋値のペア）
- 出力：確認UI経由でCSV/Sheets出力 or 既存システムAPI呼び出し
- 言語：日本語

### OUT（MVP非対象）
- 自由記述欄の意味解釈
- 複数システムへの同時登録
- リアルタイム処理（バッチで十分）
- モバイルキャプチャ（PC画面前提）

---

## 4. 成功指標

| KPI | 目標値 | 計測方法 |
|-----|--------|----------|
| 項目抽出精度 | ≥ 90%（F1） | PoC評価セットで測定 |
| 入力時間削減率 | ≥ 30% | Before/After計測 |
| ヒューマンエラー率 | ≤ 現状の1/3 | エラーログ比較 |
| オペレーター確認件数/日 | 定常化後に定義 | システムログ |
| 例外（抽出失敗）率 | ≤ 10% | PoC期間中に許容値定義 |

---

## 5. フェーズロードマップ概要

```
Phase 1: PoC      (Week 1〜4)  → 精度確認・技術選定
Phase 2: MVP      (Month 2〜3) → 入力支援として本番投入
Phase 3: 半自動   (Month 4〜6) → 高信頼項目の自動反映
Phase 4: 全自動   (Month 7〜)  → 学習ループ・監査強化
```
