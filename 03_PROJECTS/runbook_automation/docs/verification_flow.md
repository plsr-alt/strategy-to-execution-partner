# 検証フロー

## Phase 1: PoC

### 検証目的
マルチモーダルLLM（方式A）で対象帳票1種の項目抽出が実用精度に達するか確認する。

### 前提条件
- 対象帳票: [仮] 最も処理頻度の高い帳票1種
- サンプル数: 50件[仮]
- 抽出項目: 5-10項目[仮]

### 検証手順
1. サンプル画像50件を収集（実際の業務で使う画像品質）
2. 正解データを人手で作成（項目×50件のスプレッドシート）
3. Claude APIにサンプル画像を投入、JSON出力を取得
4. 正解データと突合し、項目単位の正答率を算出
5. エラーパターンを分類（読み取りミス / フォーマット崩れ / 項目の取り違え）
6. 処理時間・エラー率を記録
7. 手作業での同等処理時間を計測し、削減率を算出

### 成功条件
| # | 条件 | 基準値 |
|---|------|--------|
| 1 | 項目抽出精度 | 90%以上 |
| 2 | 処理時間 | 30秒/件以内 |
| 3 | 作業時間削減率 | 50%以上 |
| 4 | エラー率 | 5%以下 |
| 5 | 帳票カバー率 | 全項目抽出可能 |

### 判定方法
- 条件1-4をすべて満たす → **成功** → Phase2へ
- 条件1が80-90% → **条件付き成功** → プロンプト改善後に再検証
- 条件1が80%未満 → **不合格** → 方式B（OCR+LLM）で再検証

### 判定日
PoC開始から2週間後
