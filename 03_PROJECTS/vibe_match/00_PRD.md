## A. 問題・解決策・差別化

### 問題
- 既存マッチングアプリは写真の"見た目の好み"が主導 → スワイプ疲れ／会話開始率低い
- プロフィール文章は作り込み差が大きい
- 「並んだ時の雰囲気が合う/映える」を探すのが難しい（言語化できない）

### 解決策
- 顔写真から「顔の類似」ではなく「顔タイプ/雰囲気タグ」を推定
- **統一感マッチ** / **補完マッチ** の2モードで推薦
- ユーザーへは「印象タグ」と「相性理由」を提示し、納得感を担保

### 差別化（Pairs/Tinder/Bumble等との軸）
- 「外見点数」でも「似てる判定」でもなく、**雰囲気の相性に特化**
- 2つの探索モード（統一感/補完）をプロダクトUIに組み込み
- 説明可能性（なぜこの推薦か）を必須要件として設計
- 顔照合・有名人類似・ユーザー間類似度表示を一切しないガードレール

## C. ユーザーストーリー / JTBD

1. **恋活：直感の言語化** — 写真の印象を言葉にできない → 印象タグで自信を持って相手探し
2. **婚活：相性の良い"系統"に絞りたい** — 闇雲なスワイプ → 統一感/補完で時間対効果よくマッチ
3. **会話の入り口が欲しい** — 初手メッセージに困る → 推薦理由が会話ネタに
4. **炎上しない安心** — 見た目採点や似てる判定は不快 → ルッキズム非助長設計
5. **好みを学習してほしい** — いいね/スキップが反映 → 使うほど精度向上

## D. ペルソナ

### P1：恋活女性（24歳・都内・会社員）
- 目的：「会ってみたい雰囲気」重視
- 課題：スワイプ疲れ
- 価値：印象が言語化、合う系統が見つかる
- 懸念：見た目採点・写真悪用

### P2：恋活男性（28歳・関西・IT）
- 目的：マッチ後に会話が続く相手
- 課題：何が刺さるかわからない
- 価値：印象タグ→プロフィール改善ヒント
- 懸念：AI診断が雑だと信用しない

### P3：婚活（女性32 or 男性34・首都圏）
- 目的：短期間で結婚を見据えた相手
- 課題：条件検索はできるが「雰囲気の相性」が外れる
- 価値：統一感/補完で並んだ時の違和感を減らす
- 懸念：AIが偏見を生む・個人特定の不安

## E. ユーザーフロー（MVP）

```
登録 → プロフィール入力 → 写真アップロード → AI診断（印象タグ生成）
  → 探索（統一感/補完モード） → いいね/スキップ → マッチ → チャット
```

### 主要ユースケース
- U1：写真アップ→印象タグ診断→推薦閲覧
- U2：統一感/補完モード切替→推薦更新
- U3：推薦にいいね→相互でマッチ→チャット開始
- U4：診断微妙→写真追加/撮り直し→再診断
- U5：通報/ブロック

## F. 機能要件

### MUST（MVP必須）
- アカウント/認証（電話番号 or Apple/Google）
- プロフィール（基本属性）
- 写真（1-3枚、顔検出・品質チェック）
- AI診断（印象タグTopN + 信頼度 + 不確実性UX + 禁止推定フィルタ）
- レコメンド（統一感/補完モード切替、相性スコア、理由）
- いいね/スキップ/マッチ/マッチ一覧
- チャット（テキストのみ、通報/ブロック）
- 収益化（プレミアム枠 1-2点 + ブースト）
- 管理コンソール（最低限）

### SHOULD（β）
- 好み学習（いいね/スキップ→レコメンド反映）
- 会話支援（スターター自動生成）
- 本人確認/年齢確認強化（eKYC）
- 画像送信（モデレーション付き）
- 不正対策（多重アカウント・スパム・BOT検知）
- 多様性制御

### COULD（正式版以降）
- 価値観質問＋タイプ相性統合
- デートプラン提案
- グループ/イベント機能
- オフライン本人確認連携

## H. AI要件

### 入力
- 顔写真 1-3枚（JPG/PNG/WebP、最大5MB/枚）
- 品質ゲート：顔20%以上、正面±20度、明るさ閾値、単独人物

### 出力
- 印象タグ Top5: `[{id, label, probability(0-1)}]`
- confidence: 0-100
- compatibility: `{mode, score(0-100)}`
- explanation: 理由3点

### 信頼度しきい値
| confidence | 対応 |
|-----------|------|
| ≥70 | 通常表示 |
| 40-69 | 注意表示 + 追加写真導線 |
| <40 | 診断保留 + 撮り直し必須 |

### 禁止推定（必須）
人種/民族、国籍、健康状態、障害、妊娠、性的指向、宗教、政治、年齢断定、個人識別、有名人類似

## I. マッチング/レコメンド要件

### 統一感マッチ（Unified）
- 両者の印象タグ分布が近い（タイプ距離が小さい）
- "並んだ時に絵が整う"方向

### 補完マッチ（Complement）
- 主要タグが相補的（事前定義の相性行列）
- "対比で映える/バランスが良い"方向

### スコアリング
```
最終スコア = w1*タイプ相性 + w2*好み学習 + w3*行動適合 + w4*新規性/多様性 + w5*制約
```
