## 1. AI生成パラメータ・工夫

### 1.1 FLUX.2 プロンプト最適化

**テーマ: AIグラビア**

#### 効果的なプロンプト例
```
[成功例]
"8K magazine-quality portrait photo, professional model, natural light,
studio setting, [specific pose], realistic skin texture, high fashion aesthetic"

[失敗例] ❌
"pretty girl" → 顔の破綻・不自然な構図多発
"anime girl" → アニメ混在・形式不統一
```

**工夫**:
- 「8K」「magazine-quality」で品質指定
- 「professional model」で見た目統一
- 「natural light」で自然な陰影
- 「realistic skin texture」で破綻低減

#### LoRA 学習ポイント
- [ ] 学習用データ: 100-200枚の良質画像セット必須
- [ ] 学習ステップ: 500-1000ステップが目安（過学習回避）
- [ ] 評価: 定期的に生成テストで品質確認

**記録**: パラメータセット別の成功率をスプレッドシートで追跡

## 2. プラットフォーム運用のコツ

### 2.1 pixivFANBOX

#### ハマりポイント
**❌ Issue 1: 支援者が増えない**
- 原因: FANBOX リンクの露出不足
- 解決:
  - pixiv プロフ欄に FANBOX リンク **太字で配置**
  - pixiv 投稿説明文に毎回「続きはFANBOXで」記載
  - 予約投稿で自動拡散（投稿時刻を16:00に統一）

**❌ Issue 2: 無料コンテンツをどう配置するか**
- 原因: FANBOX 有料化で閲覧減少
- 解決:
  - pixiv で無料投稿（50-70%を公開）
  - FANBOX で 限定版（高解像度 + 追加画像）を配置
  - 「FANBOX限定20枚」という表記で価値訴求

#### 成功パターン
✅ **「3日無料キャンペーン」は高効果**
- 初期ユーザー 5-10人が、その後継続支援に移行
- 継続率: 70-85%

✅ **リクエスト権は必須機能**
- Premium プラン（3,000円）の差別化要素
- 月1回のオーダーメイド対応で粘着性向上

---

### 2.2 BOOTH

#### ハマりポイント
**❌ Issue 1: 商品が埋もれる**
- 原因: 新着順ソートで数日で埋もれ
- 解決:
  - 毎週 木曜に新商品を 2-3個リリース（継続的新着化）
  - タグを **最大10個** まで活用（「AI生成」「グラビア」「イラスト」等）
  - 商品説明は **目玉焼き画像（インパクト重視）** を先頭に配置

**❌ Issue 2: 価格が高く見える**
- 原因: 初心者は 800円が心理的抵抗
- 解決:
  - **初回特価**: 最初の 1-2セットは 600円（後に値上げ）
  - **バラ売り**: 5枚セットを 200-300円で提供（低価格エントリー）
  - セット内容を「20-30枚」と明記（単価感を下げる）

#### 成功パターン
✅ **サムネイルが全て**
- 第1画像が検索結果で表示される唯一の要素
- 高解像度 + インパクトある画像を必須
- テンプレート化: サムネイル 300×300px の AI生成テンプレを事前準備

✅ **セット販売 + バラ売り組合わせ**
- セット（800円）× セット買い層
- バラ売り（200円）× ライト層
- 両者を満足させる価格体系

---

### 2.3 DLsite

#### 注意点
- **申請期間**: 1-2週間かかる（早めの申請推奨）
- **AI カテゴリ**: 必ず「AI 専用ブース」に登録（通常の同人カテゴリは避ける）
- **画像品質**: 成人向けは解像度 1200×1600 以上推奨

#### 成功パターン
✅ **複数セットの同時リリース**
- DLsite は「新着」露出が限定的
- 5-10セットを同時投稿で、ランキング入り可能性向上

## 3. SNS 発信・プロモーション

**❌ Issue 1: ランキング入りしない**

- 解決:

```
スライド: 20-30枚
```

理由: 20-30枚スライドは pixiv ランキングロジックで有利（１投稿 = １作品扱い）

## 5. コンテンツ制作の効率化

### 5.1 バッチ処理化

#### 発見: 週単位でまとめて制作するほうが 3倍効率

```
❌ 非効率: 毎日 1セット制作
→ セットアップ時間 × 7回 = 無駄

✅ 効率的: 週1回 まとめて 3-4セット制作
→ セットアップ時間 × 1回 + バッチ生成で時間短縮
```

**実装方法**:
- 月曜朝: プロンプト 5-10個を事前準備
- 火曜-木曜: AI バッチ生成 + 並列画像処理
- 金曜: 品質チェック + パッケージング
- 土曜: アップロード + SNS 投稿

---

### 5.2 テンプレート化

#### 効果: 同じ工程を繰り返さない

```
作成済みテンプレート:
✅ サムネイル作成テンプレ（Photoshop）
✅ 商品説明文テンプレ（BOOTH/FANBOX）
✅ ZIP パッケージング フォルダ構成
✅ メタデータ埋め込みスクリプト（画像に自動付与）
```

**効果**: 新規コンテンツ制作時の手作業 50% 削減

## 6. 法的・規約対応ノウハウ

### 6.1 AI 表記のベストプラクティス

#### 全プラットフォーム統一表記

```
【商品説明文に追加】
"このコンテンツは AI（FLUX.2）を使用して生成されています。
生成される内容は 生成モデルの出力のため、
同じプロンプトでも異なる結果が生成される可能性があります。"
```

**理由**:
- BOOTH 規約対応（必須）
- pixivFANBOX ユーザー信頼構築（推奨）
- DLsite AI ブース（自動表記される場合あり）

---

### 6.2 著作権対応チェックリスト

```
□ プロンプトに実在キャラ名を含めない
□ 実在人物の名前・顔特徴を指定しない
□ 背景素材は著作権フリー素材から選定
□ 生成画像の再販売・改変を禁止と明記
□ LoRA 学習データに版権素材を含めない
```
