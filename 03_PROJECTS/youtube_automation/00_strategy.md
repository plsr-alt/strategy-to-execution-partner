# YouTube動画自動化 — 戦略＆ロードマップ

## 1. 結論

**A案「非属人×オリジナル解説」路線** を推奨。理由：

- 規約耐性が最も高い（量産判定を回避）
- 外注・自動化がしやすい（台本→音声→映像→編集が完全分離可能）
- 既に `video_auto_edit` パイプラインがあるので、上流（企画→台本→素材）を繋げるだけ

---

## 2. 現状の強み（すでに持っているもの）

| 資産 | 用途 |
|------|------|
| `video_auto_edit/` パイプライン | 無音カット→テンポ調整→字幕→書き出し |
| WSL + Groq 環境 | 台本生成・市場調査のLLM基盤 |
| CrewAI セットアップ | エージェント型の調査・生成ワークフロー |
| Claude Code | 戦略判断・レビュー・品質管理 |

→ **足りないのは「企画→台本→音声→映像素材」の上流パイプライン**

---

## 3. 全体アーキテクチャ（目標状態）

```
[企画エンジン]        ← Groq/Claude: トレンド分析→テーマ選定→構成決定
    ↓
[台本生成]           ← Groq (LLM): 台本→フック→CTA生成
    ↓
[音声生成]           ← ElevenLabs / VOICEVOX / Google TTS
    ↓
[映像素材生成/収集]   ← Pexels API / AI画像生成 / 図解スライド自動生成
    ↓
[動画組み立て]        ← ffmpeg + video_auto_edit パイプライン
    ↓
[サムネイル生成]      ← AI画像 + テンプレ合成
    ↓
[メタデータ最適化]    ← タイトル・説明・タグのSEO最適化
    ↓
[アップロード]        ← YouTube Data API v3 で自動投稿
```

---

## 4. ツール選定（2026年現在）

### Tier 1: 即採用（無料〜低コスト）

| 工程 | ツール | コスト | 理由 |
|------|--------|--------|------|
| 台本生成 | **Groq (Llama 3.1)** | 無料 | 既にWSLに構築済。高速 |
| 音声生成 | **VOICEVOX** | 無料 | 日本語特化。商用利用可。OSS |
| 映像素材 | **Pexels API** | 無料 | 高品質ストック映像。API制限緩い |
| 図解生成 | **Python (matplotlib/Pillow)** | 無料 | スライド風図解を自動生成 |
| 動画編集 | **video_auto_edit（自作）** | 無料 | 字幕・カット・テンポ調整 |
| サムネイル | **Pillow + テンプレ** | 無料 | テンプレ画像に文字合成 |
| SEO | **YouTube Data API** | 無料 | メタデータ設定・投稿自動化 |

### Tier 2: 品質向上時に検討

| 工程 | ツール | コスト | 理由 |
|------|--------|--------|------|
| 音声（高品質） | **ElevenLabs** | $5〜22/月 | 英語なら圧倒的。日本語もOK |
| 映像生成 | **Pika / Runway** | $8〜15/月 | AIビデオ生成。リアル系素材 |
| 台本改善 | **Claude API** | 従量 | 品質重視の台本校正 |
| サムネイル | **Canva Pro** | $13/月 | テンプレ豊富。Magic Studio |

## 6. 30日ロードマップ

### Phase 0: 準備（Day 1-3）
- [ ] チャンネルコンセプト決定（ジャンル×ターゲット×差別化軸）
- [ ] 競合分析：伸びてる動画20本を分解
  - 構成・尺・導入5秒・離脱ポイント・サムネ・タイトル
- [ ] YouTube Data API のセットアップ
- [ ] VOICEVOX インストール・音声テスト

### Phase 1: パイプライン構築（Day 4-7）
- [ ] 台本生成スクリプト実装（Groq LLM）
  - テーマ → アウトライン → 台本（フック + 本題 + CTA）
- [ ] 音声生成スクリプト（VOICEVOX API or TTS）
- [ ] 映像素材収集スクリプト（Pexels API）
- [ ] サムネイル自動生成
- [ ] 全工程を繋ぐ `youtube_pipeline.py` 作成

### Phase 2: テスト投稿（Day 8-14）
- [ ] 8本作る（"型"を固定して検証速度を上げる）
- [ ] 投稿 → CTR・維持率・平均視聴時間をモニタリング
- [ ] 導入5秒 + サムネイルを A/Bテスト

### Phase 3: 最適化（Day 15-30）
- [ ] 週2-3本ペースで投稿
- [ ] パフォーマンスデータから型を改善
- [ ] 収益化条件の進捗確認（1000登録 + 4000時間 or 1000万Shorts再生）

## 9. リスクと対策

| リスク | 対策 |
|--------|------|
| 収益化停止（量産判定） | 台本に独自解釈・分析を必ず入れる。構成のバリエーションを持つ |
| AI音声がバレて信頼低下 | YouTube Studioで「AI生成」を開示。品質の高い音声を使用 |
| ネタ切れ | トレンドAPI + 競合分析の自動化で常にネタをストック |
| 競合過多 | ニッチ×ニッチの掛け算で差別化（例：AI×節約、テック×地方暮らし） |
| 規約変更 | 収益を広告だけに頼らない。アフィ・自社商品・コンサルに分散 |
