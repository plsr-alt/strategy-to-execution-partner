## 2. 5モジュール責務定義

| # | モジュール | 責務 | MVP | V1 | 依存 |
|---|-----------|------|-----|-----|------|
| 1 | **インジケーター計算エンジン** | ボリバン・RSI・ATR・移動平均の計算・キャッシング | ✓ | ✓ | Base |
| 2 | **エントリー判定ロジック** | ボリバン接触 + RSI判定の複合フィルター | ✓ | ✓ (ADX追加) | Module 1 |
| 3 | **ポジション管理・資金管理** | SL・TP設定、固定ロット計算（2%）、最大保有期間管理 | ✓ | ✓ (動的SL) | Module 2 |
| 4 | **終了条件・クローズロジック** | TP達成・SL・強制クローズの判定・実行 | ✓ | ✓ | Module 3 |
| 5 | **ログ・統計・アラート** | 取引履歴記録、勝率/PF/MaxDD計算、警告通知 | △（簡易） | ✓（フル） | Module 4 |

**モジュール責務説明**:

**Module 1：インジケーター計算エンジン**
ボリバン・RSI・ATR・移動平均の4指標を足ごとに計算し、次のロジック層に供給。計算バグ（過去データ参照ミス・インデックスずれ）が全体精度に直結するため、単体テスト必須。

**Module 2：エントリー判定ロジック**
ボリバン上下限接触 + RSI 30/70交差の複合フィルターで売り・買いエントリー判定。「ボリバン + RSI」で ダマシ低減。判定甘化でレンジ内無駄エントリー増加 → パラメータ調整後は統合テスト必須。

**Module 3：ポジション管理・資金管理**
エントリーから SL値・TP値・ロット数を計算して確定。固定リスク2%方式による複利効果が複数月で累積 → ロット計算エラーは致命的。Lot動的変化を確認するテストケース必須。

**Module 4：終了条件・クローズロジック**
TP・SL・強制クローズ(72h)のいずれかでポジションをクローズし P&L確定。未決済ポジション発生で統計計算破綻 → バックテストログで「未決済ポジション 0個」確認必須。

**Module 5：ログ・統計・アラート**
取引ごとに履歴（時刻・価格・SL・TP・クローズ価格・P&L）を記録し、勝率・PF・MaxDD・連敗回数を計算。MVP は簡易版（ターミナル出力）で充分。V1 で CSV化＋グラフ化を実装。

## 5. 4種テスト計画

### 5.1 単体テスト（Unit Test）

| # | テスト対象 | テストケース | 期待結果 | 合格基準 |
|---|----------|----------|---------|--------|
| 1 | ボリバン計算 | 期間20・σ2 での上下限値 | MT4標準値と一致 | 誤差 0pips |
| 2 | RSI計算 | 14期間での値 | RSI値 0～100範囲内 | 範囲内 + 精度小数点2位 |
| 3 | ATR計算 | 14期間でのボラティリティ | ATR>0 / 前足より過度に増減しない | 物理的妥当性確認 |
| 4 | ロット計算 | 口座100万・SL20pips→Lot計算 | Lot=0.5を確認 | 誤差なし |
| 5 | エントリー判定 | BB接触+RSI30→売りシグナル | True を返す | 判定ロジック正確性 |
| 6 | SL/TP計算 | ATR=20→SL=40・TP=20MA値 | 妥当範囲内 | 計算根拠との整合性 |

**注記**：単体テストは各モジュール開発直後に実施。計算精度が全体品質を左右するため、特に Module 1・3 は念入り。

### 5.2 統合テスト（Integration Test）

| # | テスト対象 | テスト手順 | 期待結果 | 合格基準 |
|---|----------|---------|---------|--------|
| 1 | Module 1→2 データ流通 | 指標値計算後、エントリー判定へ受け渡し | エントリー判定が計算値を正しく参照 | シグナル発火が期待通り |
| 2 | Module 2→3 ポジション確定 | エントリーシグナル発火 → SL・TP・Lot確定 | ポジション構造が正確に作成 | P&L計算で矛盾なし |

### 5.3 バックテスト（Strategy Test）

#### 段階1：粗検バックテスト（Open prices only）

- **目的**: 概略検証・高速判定
- **期間**: 直近12ヶ月 USD/JPY 1H足
- **テストモデル**: Open prices only
- **処理時間**: 3～4日以内
- **期待結果**:
  - 勝率 > 50%
  - Profit Factor > 1.3
  - Max DD < 20%
- **GO判定**: 3つ全て満たす → 精細検証へ進む
- **NO-GO判定**: いずれか1つ未達 → パラメータ見直し

#### 段階2：精細検証バックテスト（Every tick based on real ticks）

- **目的**: 最高精度検証・スリッページ等の実物理を再現
- **ツール**: Tick Data Suite（精度99.9%）
- **処理時間**: 48～72時間（土日含めて5日）
- **テストモデル**: Every tick based on real ticks
- **期待結果**:
  - 粗検比で -20～30% の成績低下を許容
  - 月利 ≥ 3.0%
  - 勝率 ≥ 50%
  - Max DD ≤ 20%
  - Profit Factor ≥ 1.3
- **GO判定**: 全て満たす AND ウォークフォワード合格 → リアル稼働準備
- **NO-GO判定**: いずれか未達 → パラメータ見直し
