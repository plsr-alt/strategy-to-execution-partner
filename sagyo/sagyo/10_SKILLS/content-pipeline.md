# Skill: content-pipeline

## 目的

アイデアや素材から公開可能なコンテンツ（記事・SNS投稿）を一気通貫で制作し、効果測定まで管理する。

## トリガー

- 「記事にして」
- 「投稿作って」

## 入力

ユーザーからの指示（任意の組み合わせ）:
- テーマ / キーワード
- ターゲット読者
- 媒体指定（note / X / LinkedIn / ブログ等）
- トーン（専門的 / カジュアル / 煽り控えめ 等）
- 文字数目安
- 参考URL・素材

指定がない項目は過去の好み（`01_MANUAL/prefs_and_principles.md`）またはデフォルト値で仮決めする。

## 参照ファイル/データ

| ソース | 用途 |
|--------|------|
| `04_RESEARCH/*.md` | テーマに関連する調査データ・ファクト |
| `05_CONTENT/content_backlog.md` | 既存のネタ帳との重複確認・紐付け |
| `05_CONTENT/analysis/*.md` | 過去コンテンツの効果データ（あれば） |
| `01_MANUAL/prefs_and_principles.md` | 文体・トーンの好み |
| `02_STRATEGY/*.md` | ブランド方針・ポジショニング |

## 手順

### Step 1: 情報収集
- `04_RESEARCH/` から関連する調査ログを検索
- 不足があればWeb検索で補完し、結果を `04_RESEARCH/` に保存
- 一次情報・データを優先、主張にはソースを付ける

### Step 2: 論点整理
- 集めた情報から「読者にとっての価値」を1文で定義する
- 主張（=結論）を先に決め、裏付けとなるポイントを3〜5個に絞る
- 競合コンテンツとの差別化ポイントを明確にする

### Step 3: 構成作成
- 見出し構成をアウトラインで提示し、ユーザー確認を取る
- 基本構成:
  1. フック（課題提起 or 意外性）
  2. 主張
  3. 根拠 × 3〜5
  4. 反論への対処（任意）
  5. まとめ・CTA

### Step 4: 下書き
- アウトライン承認後、本文を一気に書き上げる
- 1段落＝1メッセージの原則。冗長な前置きは削る
- 保存: `05_CONTENT/drafts/YYYY-MM-DD_タイトル_v1.md`

### Step 5: 推敲
- 以下の観点でセルフレビューし修正:
  - 結論ファースト: 各段落の1文目だけ読んで意味が通るか
  - 削れる文: なくても意味が変わらない文は削除
  - 具体性: 抽象語を数字・固有名詞に置換
  - リズム: 長文と短文の交互配置
- 修正後 `_v2.md` として保存

### Step 6: 公開文面作成
- 本文からSNS用の短文を3種作成:

| 媒体 | 仕様 |
|------|------|
| **X（Twitter）** | 140字以内。フック重視。ハッシュタグ1〜2個 |
| **note** | 冒頭紹介文200字程度。記事URLへ誘導 |
| **LinkedIn** | ビジネストーン。学び・示唆を強調。300字程度 |

- CTA（行動喚起）を本文末尾・SNS各文面に含める

### Step 7: 効果測定と次企画
- 公開後、効果データを `05_CONTENT/analysis/YYYY-MM-DD_タイトル_analysis.md` に記録
- 記録項目: PV / いいね / シェア / コメント / CVなど取得可能な指標
- 結果から次のコンテンツ企画案を1〜2本提案し `content_backlog.md` に追加

## 出力

### 成果物一覧

| 成果物 | 内容 |
|--------|------|
| **本文** | 完成原稿（見出し・本文・CTA含む） |
| **SNS短文 × 3** | X / note / LinkedIn 用の投稿文面 |
| **CTA** | 読者に取ってほしいアクション（本文末尾+SNS各文面に組み込み） |
| **次ネタ案** | 今回の記事から派生する企画1〜2本 |

### 出力フォーマット

```markdown
# {タイトル}

{本文}

---

## SNS投稿文面

### X
{140字以内}

### note
{200字程度の紹介文}

### LinkedIn
{300字程度}

---

## 次のコンテンツ案
1. {ネタ案1} — {狙い}
2. {ネタ案2} — {狙い}
```

## 保存先

| ファイル | パス |
|---------|------|
| 下書き（v1, v2...） | `05_CONTENT/drafts/YYYY-MM-DD_タイトル_v{n}.md` |
| 効果測定 | `05_CONTENT/analysis/YYYY-MM-DD_タイトル_analysis.md` |
| バックログ更新 | `05_CONTENT/content_backlog.md` にステータス反映 |
| 調査素材 | `04_RESEARCH/YYYY-MM-DD_テーマ名.md`（新規調査時） |
