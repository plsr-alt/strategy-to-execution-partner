## テンプレート（コピー用）

```
================================================================
【スライド番号：　　　　】

■ Title（タイトル）:
〔スライドの主題。短く。主張ではなく主題。〕

■ Message Line（メッセージライン / 1文で）:
〔このスライドで読み手に伝えたい一番のポイント。動詞を含む完全な1文。〕
〔例：施策Aを今期中に導入することで、年間コストを2,000万円削減できる。〕

■ Sub Message（サブメッセージ / 任意）:
〔メッセージラインの補足フレーズ。なければ空欄でよい。〕

■ Body Type（ボディ型）:
  □ 表（Table）      ← 複数項目の比較・整理
  □ グラフ（Chart）  ← 数値の傾向・割合・推移
  □ 概念図（Concept）← 関係性・構造・フロー・プロセス
  □ テキスト（Text） ← 定性的説明・箇条書き

■ Body Content（ボディ内容 / 草案）:
〔メッセージラインを補完・裏付けするコンテンツ。〕
〔表の場合：行数×列数と列名・主要数値を記載〕
〔グラフの場合：軸名・データ系列・レンジを記載〕
〔概念図の場合：構成要素と矢印の方向・ラベルを記載〕
〔テキストの場合：箇条書き5行以内で記載〕

■ Key Numbers / Assumptions（キー数値・前提条件）:
〔メッセージを支える根拠数値や前提条件を記載。〕
〔例：調査期間：2024年4月〜2025年3月 / 対象：国内XX社〕

■ Footnote（注釈）:
〔前提条件・用語定義・補足説明。スライド下部に小さく表示。〕
〔不要な場合は「なし」と記入〕

■ Source（出所）:
〔データ・グラフ・引用の出典。「出所：〇〇」の形式で記載。〕
〔不要な場合は「なし」と記入〕

■ Speaker Notes（話す順・補足 / 任意）:
〔プレゼン時に口頭で補足する内容。スライドに書かない情報。〕

■ Layout Notes（レイアウト指示）:
〔整列・余白・強調すべき箇所の指示〕
〔例：「2,000万円」を赤太字で強調 / 余白30%確保 / ボディは左寄せ〕

================================================================
```

## 記入例：新規作成

```
================================================================
【スライド番号：2 / 3】

■ Title（タイトル）:
施策A：コスト削減効果の根拠

■ Message Line（メッセージライン）:
施策Aを今期中に導入することで、年間コストを2,000万円（現状比30%）削減できる。

■ Sub Message（サブメッセージ）:
○○社の類似事例では6ヶ月で効果が顕在化している。

■ Body Type（ボディ型）:
  ✅ 表（Table）

■ Body Content（ボディ内容）:
3行×4列の比較表
列名：施策名 / 初期コスト / 年間削減額 / 回収期間
行1：施策A / 500万円 / 2,000万円 / 3ヶ月
行2：施策B / 1,200万円 / 1,500万円 / 10ヶ月
行3：施策C / 300万円 / 600万円 / 6ヶ月
→ 施策Aの行を強調色（青背景）で示す

■ Key Numbers / Assumptions（キー数値・前提）:
- コスト削減試算は2024年度の実績費用をベースに試算
- 施策Bは外部ベンダー費用を含む

■ Footnote（注釈）:
※回収期間は初期コスト÷月間削減額で算出。人件費削減分は含まない。

■ Source（出所）:
出所：社内コスト管理資料（2025年1月版）/ ○○社導入事例レポート（2024年）

■ Speaker Notes（話す順）:
1. まず施策Aの削減額（2,000万円）を強調
2. 他施策と比較して初期コストが低く回収が早い点を説明
3. ○○社での成功事例に触れてリスク低減を示す

■ Layout Notes（レイアウト指示）:
- 施策Aの行を青背景で強調
- 「2,000万円」をメッセージライン内で太字
- 表の上下に十分な余白
- 出所は左下、フォント8pt

================================================================
```

## 記入例：添削後（Before → After）

### Before（添削前）
```
タイトル：DX推進状況について
本文：
・プロジェクトAは進行中
・プロジェクトBは来月リリース予定
・リソースが足りていない
・予算超過が発生している
・ステークホルダーとの調整が必要
```

### After（添削後テンプレ）
```
================================================================
【スライド番号：1】

■ Title：DX推進プロジェクト状況

■ Message Line：
プロジェクトAは予定通り進行中だが、Bはリソース不足により来月リリースが遅延するリスクがある。

■ Sub Message：
今週中に追加リソースの承認が必要。

■ Body Type：
  ✅ 表（Table）

■ Body Content：
2行×4列
列名：PJ名 / ステータス / 懸念点 / 必要アクション
行1：PJ-A / ✅ 予定通り / なし / —
行2：PJ-B / ⚠️ 遅延リスク / リソース不足 / 追加要員1名 今週中承認

■ Key Numbers：なし

■ Footnote：※リリース予定日：PJ-A 3/31、PJ-B 4/15（現状）→ 5/15（リスクシナリオ）

■ Source：なし

■ Speaker Notes：
PJ-Bの遅延リスクを中心に説明し、意思決定（リソース承認）を促す。

■ Layout Notes：
PJ-Bの行を赤背景で強調 / 「今週中承認」を太字 / 余白確保

================================================================
```

## クイックリファレンス（コピペ用チェックリスト）

```
【提出前チェック】

目的・読み手
  □ 読み手は初見で理解できるか？
  □ 読んだ後に取るべきアクションが明確か？

メッセージライン
  □ 1文（完全文）で書かれているか？
  □ 「で、何が言いたいの？」が残らないか？
  □ タイトルと役割が分かれているか？

ボディ
  □ メッセージラインを直接裏付けているか？
  □ 最適なボディ型（表/グラフ/概念図/テキスト）を選んでいるか？
  □ 情報量が多すぎないか？（箇条書き5行以内）

注釈・出所
  □ データに出所が記載されているか？
  □ 根拠なし断定になっていないか？

見た目
  □ 余白が十分か？（目安30%以上）
  □ 強調箇所は1〜2箇所のみか？
  □ 整列が取れているか？
  □ フォントが統一されているか？
```
