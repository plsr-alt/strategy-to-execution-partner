# Accenture Slide Standard Skill

アクセンチュア公式ブログ「PowerPointスライド作成の秘訣」を根拠とした、スライドの新規作成・添削/リライトを同一基準で行うスキル。

根拠記事: https://www.accenture.com/jp-ja/blogs/japan-careers-blog/powerpoint

---

## 提供価値

このスキルは **2つのモード** を提供する。

| モード | 概要 |
|--------|------|
| **新規作成モード** | ゼロから「Accenture基準」で1枚〜複数枚を設計→文章/図表指示まで作る |
| **添削/リライトモード** | 既存スライドのテキスト/構成を投入すると、診断→修正案→完成稿を返す |

どちらも **同一の原則・チェックリスト**（→ `references/accenture-slide-principles.md`）で回す。

---

## モード別の使い方

### モード1：新規作成

**起動フレーズ例**
```
【新規スライド作成】
- 目的：○○提案の承認を得る
- 読み手：部長・意思決定者
- 状況：週次レビューで5分のプレゼン
- 伝えたいこと：△△により、コストを30%削減できる
- 枚数：3枚（概要・根拠・次アクション）
```

```
【新規スライド作成】
- 目的：チームへの施策展開
- 読み手：現場メンバー（技術者10名）
- 素材：KPIデータ（別途貼付）
- 枚数：1枚
```

**詳細ワークフロー** → `references/creation-workflow.md`

### モード2：添削/リライト

**起動フレーズ例**
```
【スライド添削】
以下のスライドを診断・リライトしてください。

タイトル：DX推進状況について
本文：
・プロジェクトAは進行中
・プロジェクトBは遅延
・来月にはリリース予定
・リソースが足りていない
```

```
【スライド添削】
このスライドを改善してください。
文脈：経営会議向けの月次報告資料

＜現状スライド＞
タイトル：売上について
・先月比 +5%
・顧客数増加
・一部地域で苦戦
```

**詳細ワークフロー** → `references/review-workflow.md`

## 出力フォーマット（共通）

スライド1枚ごとに以下の順で出力する。

```
---
【スライド N】

■ Title（タイトル）
〔スライドの主題〕

■ Message Line（メッセージライン）
〔このスライドで読み手に伝えたい一番のポイント。1文。〕

■ Body Type（ボディ型）
□ 表 / □ グラフ / □ 概念図 / □ テキスト  ← 選択したものに ✓

■ Body Content（ボディ案）
〔メッセージラインを補完する内容。表・グラフなら構造を示す。〕

■ Key Numbers / Assumptions（キー数値・前提）
〔根拠となる数字や前提条件〕

■ Footnote（注釈）※必要な場合
〔前提条件・用語定義・補足説明〕

■ Source（出所）※必要な場合
〔データ・引用元の出典〕

■ Speaker Notes（話す順・補足）※任意
〔プレゼン時の補足説明〕

■ Layout Notes（レイアウト指示）
〔整列・余白・強調すべき箇所の指示〕
---
```

## 不足情報の確認優先順位

以下の情報が不足している場合、この順番で質問する。

1. **目的** — このスライドは何のために作るか？（承認/共有/説明/決定）
2. **読み手** — 誰が見るか？（役職・専門知識・初見か否か）
3. **意思決定** — 読んだ後に読み手に何をしてほしいか？
4. **1枚の結論** — このスライド1枚で言いたいことは何か？（メッセージライン）
5. **根拠データ** — 結論を裏付ける事実・数値・調査結果はあるか？

## やってはいけないこと

| NG | 理由 |
|----|------|
| スライド作成を目的にする | 目的はコミュニケーション。「作ること」は手段 |
| 1枚に複数のメッセージを詰め込む | 「で、何が言いたいの？」になる |
| メッセージラインを書かない/弱くする | 最重要コンポーネントの欠落 |
| 根拠なしに断定する | 事実・分析・提言・次アクションの裏付けが必須 |
| 出所・注釈を省略する | データの信頼性・再現性が失われる |
| 目的なき装飾・アニメーション | シンプルさの原則に反する |
| 作る前に考えない | 「考える時間」こそ本質。操作は作業 |
| 自分視点で書く | 常に初見の読み手を想定する |
