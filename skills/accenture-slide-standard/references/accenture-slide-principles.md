# Accenture スライド原則・定義

> 根拠記事：アクセンチュア公式ブログ「PowerPointスライド作成の秘訣」
> https://www.accenture.com/jp-ja/blogs/japan-careers-blog/powerpoint

---

## A. スライドの目的

### 定義
**スライド = コミュニケーションツール**

スライドは「作ること」が目的ではない。
**読み手がいて、伝えたいメッセージがある**。その橋渡しをするものがスライドである。

### スライドが伝える4要素
| 要素 | 説明 |
|------|------|
| **事実** | 調査結果・データ・現状 |
| **分析** | データから導き出された洞察・解釈 |
| **提言** | 推奨される対応策・方向性 |
| **次のアクション** | 読み手に取ってほしい具体的な行動 |

### 「目的の目的化」を防ぐ
- PowerPointでスライドを作るのはあくまで **「作業」**
- 大切なのはスライドを作り始めるまでの **「考える時間」**
- 上手く伝わらないスライドの多くは、**作成前の準備が足りていないから**
- 実践：作る前に **5mm方眼紙にメッセージ・コンテンツを下書き** してから着手する

## B. "分かり易い/見易い"の2大原則

### 原則1：シンプルさを追求する

| ルール | 具体的行動 |
|--------|-----------|
| 1スライド1メッセージ | メッセージラインは必ず1文に絞る |
| 情報を削ぐ | 「なくても伝わる」要素は全て削除 |
| 文字量を抑える | 箇条書きは5行以内を目安。説明文は短く |
| 強調は最小限 | 太字・色変えは本当に重要な1〜2箇所のみ |
| 余白を活かす | 詰め込まない。余白は「読みやすさ」のデザイン |

### 原則2：ビジュアル（見た目）にこだわる

| ルール | 具体的行動 |
|--------|-----------|
| 整列を徹底する | テキスト・図形の端を揃える |
| フォント統一 | 種類・サイズを資料全体で統一 |
| 色使いを絞る | 3色以内を基本。コーポレートカラーに準拠 |
| ボディ型を選ぶ | 表・グラフ・概念図・テキストから最適を選択 |
| スライドを「商品」として見る | 読み手が初見で美しい・読みやすいと感じるか |

> **根拠記事の言葉**：「内容は当然のこと、見た目の美しさ・分かり易さにもこだわり抜く必要がある。スライドは商品そのものだ。」

## C. スライドの構成要素（コンポーネント）

記事では「**6つ（タイトルを入れると7つ）**」の要素で構成されると述べている。

### 7コンポーネント一覧

| # | 要素 | 最重要度 | 役割 | 記載ルール |
|---|------|---------|------|-----------|
| 1 | **タイトル** | ★★★ | スライド全体の主題を示す | 短く・主題を表す名詞/フレーズ |
| 2 | **メッセージライン** | ★★★★★ | 読み手に伝えたい**最重要ポイント** | 必ず1文。動詞を含む完全な文で書く |
| 3 | **サブメッセージ** | ★★★ | メッセージラインの補足情報 | メッセージを強化する短いフレーズ |
| 4 | **ボディ** | ★★★★ | メッセージラインを補足・補完する本体 | 表/グラフ/概念図/テキストから選択 |
| 5 | **注釈** | ★★★ | 前提条件・用語定義・引用の補足 | スライド下部に小さく。必要な場合は「必ず」記載 |
| 6 | **出所** | ★★★ | データ・グラフの出典明記 | 「出所：〇〇」の形式で左下 |
| 7 | **ページ番号** | ★★ | 資料管理・参照用 | 右下に統一 |

### メッセージラインの重要性

- スライド作成で**最も頭を使う部分**
- 「このスライドを見た読み手に何を思ってほしいか」を1文に凝縮する
- メッセージラインが弱い → 「で、何が言いたいの？」という指摘が来る
- タイトルとメッセージラインは**役割が異なる**（主題 vs 主張）

**例：**
```
タイトル：国内SaaS市場の動向
メッセージライン：国内SaaS市場は2025年以降も年率15%成長が続くため、今が参入の好機だ。
```

### ボディの4つの表現型

| 型 | 使いどころ | 特徴 |
|----|-----------|------|
| **表** | 複数項目の比較・整理 | 構造的に情報を並べる |
| **グラフ** | 数値の傾向・比率・推移 | 棒/折れ線/円グラフ等 |
| **概念図（イメージ図）** | 関係性・構造・フロー | ボックス図・矢印・組織図等 |
| **テキスト** | 定性的説明・箇条書き | 簡潔な文・箇条書きで表現 |

> **選択基準**：「メッセージラインをもっとも直感的に裏付けられる型」を選ぶ。
> 数値があるならグラフ、比較があるなら表、関係性があるなら概念図を優先する。

## D. チェックリスト

スライドを確定する前に、以下を**この順番で**確認する。

### STEP 1：目的確認
- [ ] このスライドを誰が見るか明確か？
- [ ] 読んだ後に読み手に何をしてほしいか決まっているか？
- [ ] 作る前に「考える時間」を取ったか（方眼紙下書き等）？

### STEP 2：メッセージライン確認
- [ ] メッセージラインが1文で書かれているか？
- [ ] 動詞を含む完全な文になっているか？（名詞句はNG）
- [ ] 「で、何が言いたいの？」という疑問が残らないか？
- [ ] タイトルとメッセージラインで役割が分かれているか？

### STEP 3：ボディ確認（裏取り）
- [ ] ボディはメッセージラインを直接補完しているか？
- [ ] メッセージと関係のない情報が混入していないか？
- [ ] 表・グラフ・概念図・テキストから最適な型を選んでいるか？
- [ ] ボディの情報量が多すぎないか？（シンプルさの原則）

### STEP 4：注釈・出所確認
- [ ] データ・グラフに出所が記載されているか？
- [ ] 前提条件や用語定義が必要な場合、注釈に書いたか？
- [ ] 根拠なし断定になっていないか？

### STEP 5：見た目確認
- [ ] 余白が十分あるか？詰め込みすぎていないか？
- [ ] テキスト・図形が整列されているか？
- [ ] フォント種類・サイズが統一されているか？
- [ ] 強調箇所が多すぎないか？（最小限に絞る）
- [ ] 初見の読み手が美しい・読みやすいと感じる仕上がりか？
