# 新規スライド作成ワークフロー（Creation Mode）

> 原則定義 → [`accenture-slide-principles.md`](accenture-slide-principles.md)
> テンプレ → [`../templates/slide-structure.md`](../templates/slide-structure.md)

---

## 入力フォーマット

以下を指定してスキルを起動する。

```
【新規スライド作成】

- 目的：（例）△△提案の承認を得る
- 読み手：（例）部長・CFO・現場メンバー など
- 状況：（例）週次レビュー5分 / 提案書として配布 など
- 伝えたいこと：（例）施策Xによりコストを30%削減できる
- 求める意思決定：（例）来週から施策Xの実施を承認してほしい
- 必要枚数：（例）3枚（概要・根拠・次アクション）
- 素材/データ：（任意）あれば貼付
```

**省略可能な項目**: 素材/データ（なければAIが構成案を提示）
**必須項目**: 目的・読み手・伝えたいこと（この3つがないと着手不可）

---

## ワークフロー（全6ステップ）

### STEP 1：目的と読み手の確定

確認事項：
- **目的の種類**：承認獲得 / 情報共有 / 意識統一 / 意思決定 のどれか
- **読み手の解像度**：役職・専門知識レベル・初見か否か
- **読み手がスライドを見た後に取るべきアクション**

> ⚠️ 目的が曖昧な場合は、この時点でユーザーに確認する。
> 「このスライドを読んだ後に、読み手に何をしてほしいですか？」

---

### STEP 2：各スライドのメッセージラインを定義

枚数分のメッセージラインを先に確定する。

**メッセージライン定義ルール**：
- 必ず **1文（完全文）** で書く。名詞句禁止。
- 読み手視点で書く：「この人がこれを読んだら何を思うべきか」
- 全スライドのメッセージラインが **論理的につながっている** こと（ストーリーライン）

**ストーリーラインの例（3枚構成）**：
```
スライド1：〇〇事業は現在、△△という課題を抱えており対処が急務だ。
スライド2：施策Xを導入することで、課題の80%を6ヶ月以内に解消できる。
スライド3：今週中に施策Xの予算承認をいただければ、来月から開始可能だ。
```

---

### STEP 3：各スライドのコンポーネントを埋める

1枚ずつ、以下7コンポーネントを埋める（テンプレ：`templates/slide-structure.md`）。

| コンポーネント | 必須/任意 | 記載ルール |
|--------------|---------|-----------|
| Title | 必須 | スライドの主題。短く |
| Message Line | **必須** | 1文。このスライドの結論 |
| Sub Message | 任意 | メッセージの補足フレーズ |
| Body Type | 必須 | 表/グラフ/概念図/テキスト から選択 |
| Body Content | 必須 | メッセージを裏付けるコンテンツ |
| Footnote | 条件付き | 前提・用語が必要な場合は必須 |
| Source | 条件付き | データ・引用がある場合は必須 |

---

### STEP 4：ボディ型を選び、図表・データ指示を出す

**型の選択基準**：

| 状況 | 推奨ボディ型 |
|------|------------|
| 複数項目を比較したい | **表** |
| 数値の増減・割合を示したい | **グラフ**（棒/折れ線/円） |
| プロセス・構造・関係性を示したい | **概念図** |
| 定性的な説明・理由を伝えたい | **テキスト**（箇条書き） |

**図表指示の出し方**：
- 表 → 「○行×○列、列名は〜、主要数値は〜」
- グラフ → 「折れ線グラフ、X軸=期間、Y軸=売上(百万円)、データポイントは〜」
- 概念図 → 「3つのボックス（A→B→C）を左から右へ矢印でつなぐ。各ボックスに〜を記載」

---

### STEP 5：シンプル＆ビジュアル基準で整える

| チェック項目 | 基準 |
|------------|------|
| 文字量 | 箇条書きは5行以内。1行は体言止めか短文 |
| 余白 | スライド全体の30%以上は余白を確保 |
| 強調 | 太字・赤文字は1〜2箇所のみ |
| 整列 | テキスト・図形の端は揃える |
| フォント | 本文12pt以上、注釈8pt以上（9pt未満はNG） |

---

### STEP 6：最終チェックリストで検証し、完成稿を出力

`accenture-slide-principles.md` のチェックリスト（STEP1〜5）を実行。
全項目が ✅ になったら完成稿を出力する。

---
