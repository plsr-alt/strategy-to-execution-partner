# 添削・リライトワークフロー（Review Mode）

> 原則定義 → [`accenture-slide-principles.md`](accenture-slide-principles.md)
> テンプレ → [`../templates/slide-structure.md`](../templates/slide-structure.md)

---

## 入力フォーマット

```
【スライド添削】

＜文脈（任意）＞
（例：経営会議向け月次報告、承認を得たい提案書 等）

＜既存スライド＞
タイトル：〔現在のタイトル〕
本文：
・〔箇条書きや文章をそのまま貼る〕
・〔グラフ・表の場合は構造をテキストで書く〕

＜追加情報（任意）＞
- 読み手：〔例：CFO、現場メンバー〕
- 目的：〔例：コスト削減施策の承認〕
- 使いたい数値/データ：〔あれば〕
```

**省略可能**: 文脈・追加情報（ただし提供があるほど添削精度が上がる）

---

## ワークフロー（全5ステップ）

### STEP 1：問題診断

以下の観点で問題を分類・特定する。

| カテゴリ | 典型的な症状 | 診断メッセージ例 |
|---------|------------|----------------|
| **目的不明** | 何のためのスライドか分からない | 「このスライドを読んだ後に読み手に何をしてほしいか不明」 |
| **メッセージライン欠落/弱い** | タイトルがメッセージライン代わり、または曖昧な文 | 「メッセージラインが『〇〇について』という主題に止まり、結論がない」 |
| **ボディ不整合** | ボディの内容がメッセージと無関係 | 「メッセージラインと対応しないデータが混在している」 |
| **根拠・裏取り不足** | 断定しているが根拠がない | 「『効果が高い』の根拠となるデータが示されていない」 |
| **1枚に複数主張** | メッセージが2つ以上ある | 「A施策の効果とB課題の2点を1枚に詰め込んでいる」 |
| **見た目過密** | 文字・要素が多すぎる | 「箇条書きが10行あり、読み手が要点を掴めない」 |
| **出所・注釈欠落** | データの根拠が不明 | 「グラフに出所の記載がない」 |

診断は **「重大度：高/中/低」** を付けて列挙する。

---

### STEP 2：修正方針の決定

診断結果を元に、以下を明示する。

1. **削るもの**：不要な情報・冗長な説明・メッセージと無関係な要素
2. **足すもの**：メッセージライン・根拠データ・出所・注釈
3. **構造変更**：1枚を複数枚に分割する / ボディ型を変更する

**分割判断基準**：
- メッセージが2つ以上 → 枚数を増やす
- ボディの情報量が多すぎる → 詳細を別スライドか注釈へ移す

---

### STEP 3：リライト（コンポーネントを埋め直す）

`templates/slide-structure.md` のテンプレに沿って全コンポーネントを書き直す。

**メッセージライン強化の手順**：
1. 既存タイトル・本文から「最も言いたいこと」を抽出
2. 「〇〇が□□である（ため、〜すべきだ）」の形で1文化
3. 読み手が読んだ後に取るべきアクションを起点に逆算する

**ボディ型最適化**：
- メッセージが数値比較 → 表またはグラフに変換
- メッセージが手順/関係性 → 概念図に変換
- テキスト量が多い → 箇条書きを3〜5行に絞り、残りは注釈へ

---

### STEP 4："シンプル＆ビジュアル"基準で調整

| 調整項目 | Before例 | After例 |
|---------|---------|--------|
| 文字量 | 10行の箇条書き | 3行に絞り、残りを注釈へ |
| 強調 | 全行太字 | キーワード1〜2箇所のみ太字 |
| 余白 | 余白ゼロで詰め込み | 上下左右に十分な余白を確保 |
| ボディ型 | テキスト10行 | 表または概念図に変換 |

---

### STEP 5：Before/After と理由を提示

**出力構成**：
1. 診断サマリ（問題の重大度一覧）
2. 修正方針（何を削った/足した/変えた）
3. 完成稿（置き換え可能な形で出力）
4. 各修正の理由（1〜2行）

---

## 出力フォーマット

```
## 診断サマリ

| # | 問題 | 重大度 | 該当箇所 |
|---|------|--------|---------|
| 1 | メッセージライン欠落 | 高 | タイトルのみ、結論なし |
| 2 | 根拠データなし断定 | 高 | 「効果が高い」 |
| 3 | 箇条書き過多 | 中 | 本文10行 |

---

## 修正方針

- **削る**：〔例：4行目〜7行目の重複説明〕
- **足す**：〔例：コスト削減30%の数値根拠・出所〕
- **変える**：〔例：テキスト10行→表に変換〕

---

## 完成稿

■ Title：〔修正後タイトル〕
■ Message Line：〔修正後の1文メッセージ〕
■ Body Type：〔表 / グラフ / 概念図 / テキスト〕
■ Body Content：
〔修正後の内容〕
■ Key Numbers：〔根拠数値〕
■ Footnote：〔修正後の注釈 or なし〕
■ Source：〔修正後の出所 or なし〕
■ Layout Notes：〔強調・余白・整列の指示〕

---

## 修正理由

1. メッセージライン追加：タイトル「〇〇について」は主題であり結論ではなかったため、「〜すべきだ」の形で1文化した。
2. 根拠追加：「効果が高い」は断定のため、〇〇調査の数値（XX%）を根拠として付加した。
3. 箇条書き削減：10行を3行に絞り、詳細は注釈へ移動。読み手が要点を即座に掴めるようにした。
```

## よくある修正パターン集

### パターン1：タイトルがメッセージライン化している
```
【Before】
タイトル：コスト削減施策について
本文：・施策A ・施策B ・施策C

【After】
タイトル：コスト削減施策
Message Line：施策Aを今期中に導入することで、年間コストを2,000万円削減できる。
Body：施策A/B/Cの比較表（コスト・効果・実施難易度）
```

### パターン2：1枚に複数メッセージ
```
【Before（1枚）】
・売上は前期比+10%
・一方、コストも+15%増加
・新規顧客は増えているが既存顧客が離反

【After（2枚に分割）】
スライド1 Message Line：売上増加（+10%）の裏でコスト増加（+15%）が利益を圧迫している。
スライド2 Message Line：新規獲得は順調だが、既存顧客の離反が中期的なリスクとなっている。
```

### パターン3：根拠なし断定
```
【Before】
・このシステムは非常に有効だ
・導入すればコストが下がる

【After】
Message Line：〇〇社導入事例では、システム展開後6ヶ月でコストが28%削減された。
Source：〇〇社 2025年度導入事例レポート
```
