# 私の経営管理＆業務遂行システム

## このフォルダの思想

- **Claudeを経営パートナーとして使う**ための作業フォルダ
- フォルダ内のファイルがすべて「長期記憶」になる。書けば書くほどClaude の回答精度が上がる
- 外部ツール（Google Calendar等）がなくても、このフォルダだけで完結する設計

## フォルダ構成（早見表）

| フォルダ | 一言で言うと |
|---------|------------|
| `00_INBOX` | とりあえず置く場所 |
| `01_MANUAL` | 自分のルール・好み |
| `02_STRATEGY` | 経営戦略・中期計画 |
| `03_PROJECTS` | 案件ごとの管理 |
| `04_RESEARCH` | 調べもの記録 |
| `05_CONTENT` | 記事・SNS投稿 |
| `06_OPERATIONS` | 日報・週報・スケジュール |
| `07_FINANCE` | KPI・お金まわり |
| `08_PEOPLE` | 人物メモ |
| `09_TEMPLATES` | 雛形 |
| `10_SKILLS` | Claudeのスキル定義 |
| `99_ARCHIVE` | 終わったもの置き場 |

## 毎日の使い方

### 朝（5分）

Claudeに話しかける:

> おはよう

→ 前日の申し送り・プロジェクト状況・KPIを読み、今日のスケジュールと最重要タスク3つを提示してくれる。

### 日中（都度）

思いついたら一言で指示する:

> 「Issueにして：LPのコピーを来週までに修正」

> 「覚えておいて：A社の予算上限は200万」

> 「記事にして：今朝の商談で話した○○の話」

→ Claude が分類・整理・保存まで自動で行う。

### 夜（5分）

> 今日の振り返りして

→ 実行ログをもとに日報を整理し、明日への申し送りを作成。週末なら週次レビューも。

## よく使う一言コマンド

| 言うこと | Claudeがやること |
|---------|----------------|
| 「おはよう」 | 今日のスケジュール作成 |
| 「今日の予定は？」 | 同上 |
| 「Issueにして」 | タスクを整理して記録 |
| 「タスク登録して」 | 同上 |
| 「覚えておいて」 | 適切なファイルに永続保存 |
| 「記事にして」 | 記事の下書き〜SNS文面を作成 |
| 「投稿作って」 | 同上 |
| 「今日の振り返りして」 | 日報の整理・申し送り作成 |
| 「今週のまとめ」 | 週次レビュー作成 |
| 「KPI見せて」 | KPI進捗を要約して報告 |

## 外部連携がない場合の運用

Google CalendarやGitHubに繋がなくても、すべてこのフォルダで動く。

| やりたいこと | 使うファイル |
|-------------|------------|
| 予定管理 | `06_OPERATIONS/schedules/YYYY-MM-DD_schedule.md` |
| タスク管理 | `03_PROJECTS/` に案件ファイルを作る |
| KPI管理 | `07_FINANCE/KPI.md` を毎月更新 |
| メモ・記憶 | 「覚えておいて」で自動振り分け |
| 振り返り | `06_OPERATIONS/DAILY.md` `WEEKLY.md` |

外部ツールが使えるようになったら、Claudeが自動で連携を切り替える。ファイル側の運用はそのまま併用できる。

2. 「おはよう」と言う
