# Claude Code × Antigravity 活用方針・役割分担の検討

2026-02-23

現在、同じ `task` フォルダ内で「Claude Code」と「Antigravity」が共存して稼働しています。両者はトップクラスの自律型エージェントですが、**得意分野とインターフェースが明確に異なります。** 
特徴を理解し、役割を分担させることで、最強の「経営パートナー 兼 実行部隊」を構築できます。

---

## 1. 両者の特徴と強み

### 🔷 Claude Code (Anthropic)
- **インターフェース**: ターミナル (CLI)
- **モデル**: Claude 3.7 Sonnet (推論能力・コーディング能力に極めて優れる)
- **最大の強み**:
  - **超軽量・爆速**: ターミナルからシームレスに起動し、対話のテンポが速い
  - **`SKILL.md` による容易な拡張性**: ちょっとした自動化スクリプトや業務ロジック（例：`daily-schedule`）を即座にスキル化して組み込める
  - **長文コンテキストの理解**: フォルダ内の大量のMarkdown（思考ログ、ルール）を読み込んで文脈を把握するのが天才的

### 🔶 Antigravity (Google DeepMind)
- **インターフェース**: IDE拡張機能（リッチGUI・Artifactプレビュー等）
- **モデル**: Google DeepMind の最先端モデル (Gemini 派生 / 複雑なエージェント挙動に特化)
- **最大の強み**:
  - **ブラウザ操作能力**: 実際にブラウザを起動し、WebGUIの操作やスクレイピング、情報収集を自律的に行える
  - **強力なワークフローと視覚化**: `task.md` などのArtifactをリアルタイムで更新・表示する高度なUI
  - **安全な外部コマンド実行**: ローカルCLI操作や複雑な環境構築を、GUIで都度確認しながら安全に進められる

---

## 2. 推奨する役割分担（使い分け）

現状の運用ルール（`CLAUDE.md`）を活かす場合、以下のような主従関係（またはレイヤー分け）が最も効率的です。

### 🎭 役割 A：Claude Code ＝「脳・経営参謀」
主にターミナルに常駐させ、**テキスト中心の高速なやり取り**や、ルールの解釈・適用に特化させます。

- **朝と夜のルーティン**: 「おはよう」「今日の振り返りして」などのトリガーによるスケジューリングと日報作成
- **記憶の整理**: 「覚えておいて」による `agent-memory` の更新、プロジェクト情報の整理
- **壁打ち・戦略策定**: 資料の構成案づくり、ドキュメントの推敲、Markdownベースのコンテンツ作成（`content-pipeline`）

### 🛠 役割 B：Antigravity ＝「手・外部実行部隊」
主にIDEのサイドバーに常駐させ、**外部ツール・ブラウザ・コード編集が絡む重たい実作業**に特化させます。

- **Webリサーチ**: 「Zennの記事の内容を読んで実装して」「競合A社の最新IRを調べてまとめて」といったブラウザが必要な作業
- **複雑なコード実装 / デバッグ**: 実際にファイル群をまたいで大きくリファクタリングする作業、エラーを見ながらのデバッグ
- **外部出力・手順書実行**: 実際にAWSやその他のGUIコンソールをブラウザで操作させる、検証環境の構築（Runbookモードの実行部隊）

## 3. 連携のベストプラクティス（現状の最適解）

同じ作業フォルダ（`C:/Users/tshibasaki/Desktop/etc/work/task`）を共有していることが最大のメリットです。**「Claudeが考えてMarkdownに書き、Antigravityがそれを読んで実行する」**という連携が自然に成り立ちます。

### 連携例：「新しい調査をして資料を作りたい」場合

1. **[Claude Code]**: 「来週のXX社向け提案書の文脈を前提に、必要な情報収集の要件を `04_RESEARCH/requirements.md` に書き出して」と指示。
2. **[Antigravity]**: 「`requirements.md` を読んで、ブラウザで調べて結果を `04_RESEARCH/result.md` にまとめて」と指示（Antigravityがブラウザを回して調査）。
3. **[Claude Code]**: 「調査結果が出たので、いつもの私のトーン（`prefs_and_principles.md`）に合わせて構成案を作成して」と指示。

### まとめ

*   🧠 **Claude Code**: システムの「OS / 頭脳」。情報を整理し、タスクを設計し、ルールを守らせる。
*   🦾 **Antigravity**: システムの「手足」。ブラウザを駆使し、外部情報を取得し、複雑な実装や検証を行う。

今後は、**「情報の整理・設計はClaude」「外部アクセスと複雑な実装はAntigravity」**という基準で無意識に使い分けていただくと、コンテキストの無駄がなくなり、圧倒的なスピードで作業が回ります。
