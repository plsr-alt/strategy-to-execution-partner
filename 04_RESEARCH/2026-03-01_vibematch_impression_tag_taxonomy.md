# VibeMatch 印象タグ体系（Taxonomy）設計調査レポート

**作成日**: 2026-03-01
**目的**: 顔写真からAIで推定する「雰囲気タグ」の体系設計
**設計方針**: ルッキズムを助長しない／ポジティブ表現のみ／優劣をつけない

## 1. 日本で一般的に使われている顔タイプ分類

### 1-1. 調味料系（男性中心で使われることが多い）

| 分類 | 定義・印象 | 構成要素（目元・輪郭・全体バランス） |
|------|-----------|--------------------------------------|
| **塩顔** | あっさり・薄め・涼しげ | 一重〜奥二重、細め目、面長〜卵型、薄い唇、色白傾向、パーツ小さめ |
| **しょうゆ顔** | 程よい濃さ・日本的端正 | 二重、整った目鼻立ち、卵型、バランスの取れたパーツ配置 |
| **ソース顔** | 濃い・はっきり・彫り深い | くっきり二重、大きい目、彫りが深い、眉が太め、パーツが大きい |
| **マヨネーズ顔** | まろやか・親しみやすい・ふんわり | 丸顔傾向、たれ目気味、頬がふっくら、全体的に丸みのあるパーツ |
| **砂糖顔** | 甘い・可愛らしい・中性的 | 大きめの目、小さい鼻、丸み帯びた輪郭、童顔的バランス |
| **味噌顔** | 素朴・温かみ・安心感 | 穏やかな目元、広めの鼻、丸顔、全体的に柔らかい印象 |
| **酢顔** | シャープ・知的・クール | 切れ長の目、シャープな顎ライン、高い鼻筋、クール系の印象 |

**特徴**: メディアやSNSで広く認知。男性に対して使われることが多いが、女性にも応用されつつある。「濃い↔薄い」軸が基本。

| **きつね系** | 知的・妖艶・スマート | つり目、面長、切れ長、細身、知的でクールな雰囲気 |

### 1-3. 顔タイプ診断(R) 8分類（岡田実子氏提唱）

| 分類 | 定義・印象 | 構成要素（世代感 × 形状） |
|------|-----------|--------------------------|
| **キュート** | 可愛い・親しみやすい・元気 | 子供顔 × 曲線（丸い目、丸顔、曲線的パーツ） |
| **アクティブキュート** | 活発・華やか・明るい | 子供顔 × 曲線（キュートより目が大きい、パーツ強め） |
| **フレッシュ** | 爽やか・清潔感・ナチュラル | 子供顔 × 直線（すっきり目元、シンプル、直線的輪郭） |
| **クールカジュアル** | かっこいい・カジュアル・ボーイッシュ | 子供顔 × 直線（フレッシュよりシャープ、直線強め） |
| **フェミニン** | 女性らしい・柔らかい・上品 | 大人顔 × 曲線（曲線的で大人っぽい、華やか） |
| **エレガント** | 華やか・豪華・存在感 | 大人顔 × 曲線（フェミニンより派手、パーツ大きめ） |
| **ソフトエレガント** | 上品・控えめ・繊細 | 大人顔 × 中間（直線と曲線のバランス、穏やか） |
| **クール** | シャープ・知的・モード | 大人顔 × 直線（直線的パーツ、シャープ顎、切れ長目） |

**特徴**: 2軸（子供顔↔大人顔 × 曲線↔直線）で体系化されており、論理的。主に女性向けだが、男性版の応用も存在。AIでの分類と相性が良い（2軸を数値化しやすい）。

## 2. 炎上リスク分析

### 2-1. ルッキズムに抵触しうる表現

| リスクレベル | 表現・概念 | 理由 |
|-------------|-----------|------|
| **高（絶対NG）** | 「美人/イケメン度」のスコアリング | 外見に優劣をつける直接的ルッキズム |
| **高（絶対NG）** | 「○○顔ランキング」「モテ顔」 | 特定の顔タイプを上位に置く価値観の押し付け |
| **高（絶対NG）** | 肌の色・エスニシティに言及する分類 | 人種差別・差別助長 |
| **高（絶対NG）** | 体型・体重に関する分類 | ボディシェイミング |
| **高（絶対NG）** | 年齢を直接的に示す分類（老け顔/童顔を優劣で） | エイジズム |
| **中（要注意）** | 「薄い顔」「濃い顔」の二項対立 | 「薄い」をネガティブに受け取る人がいる |
| **中（要注意）** | 動物を使った比喩全般 | 「たぬき顔」等を侮辱と捉える文化圏もある |
| **中（要注意）** | 「男顔」「女顔」のジェンダー二項対立 | ノンバイナリーへの配慮不足 |
| **低（配慮必要）** | 調味料系の分類名そのもの | 定着しているが「食べ物で人を分類する」への抵抗感 |

### 2-2. 避けるべき表現・NGワード一覧

| カテゴリ | NGワード・表現 | 代替案 |
|---------|---------------|--------|
| **優劣系** | モテ顔、残念顔、普通顔、地味顔 | （使わない。全タグをフラットに） |
| **ネガティブ系** | のっぺり、薄い、地味、老けた | 「すっきり」「シンプル」「ナチュラル」「落ち着いた」 |
| **身体的特徴直接系** | でかい目、小さい目、丸い顔 | 印象語に変換（「ぱっちり」「涼しげ」「まろやか」） |
| **ジェンダー固定系** | 男顔、女顔、おじさん顔、おばさん顔 | 「クール」「ソフト」等のジェンダーニュートラル表現 |
| **人種・民族系** | ○○人っぽい、ハーフ顔 | （使わない。彫りの深さ等は別の表現で） |
| **比較優劣系** | ○○より上、整っている、崩れている | （使わない。各タグは独立した魅力として提示） |

### 2-3. 安全に使える表現のガイドライン

**原則**:
1. **全タグがポジティブ**: どのタグも「その人の魅力」として定義する。ネガティブなタグは一切作らない
2. **優劣なし**: タグ間に上下関係を設けない。UIでも均等に表示
3. **自己選択可能**: AI推定はあくまで提案。ユーザーが自分で変更・追加できる
4. **ジェンダーニュートラル**: 男女共通のタグ体系。「男性的」「女性的」ではなく「シャープ」「ソフト」等
5. **曖昧な比喩を活用**: 直接的な身体特徴ではなく、印象・雰囲気・ムードで表現
6. **複数タグ付け**: 1人に1タグではなく複数タグ（例: メイン2 + サブ3）で多面性を表現
7. **文化的配慮**: 人種・民族・肌色に関する分類は一切行わない

## 3. タグ体系の提案

### 3-1. 設計思想

```
雰囲気（Vibe） ＝ ファーストインプレッション全体の印象
  └─ 直接的な容姿の評価ではなく「その人から感じるエネルギー・ムード」

2軸モデル:
  横軸: ソフト ←→ シャープ（曲線↔直線）
  縦軸: フレッシュ ←→ リッチ（若々しさ↔成熟感）
```

### 3-2. 大カテゴリ（8タイプ）× サブタグ

| # | 大カテゴリ（Vibe Type） | 英語名 | キーワード印象 | サブタグ（3-5個） |
|---|------------------------|--------|---------------|-------------------|
| 1 | **ナチュラル** | Natural | 素朴・親しみ・安心感 | ①ほんわか ②素朴ナチュラル ③アースカラー ④のんびり系 |
| 2 | **フレッシュ** | Fresh | 爽やか・清潔感・透明感 | ①さわやか ②クリーンビューティ ③透明感 ④朝露系 |
| 3 | **キュート** | Cute | 可愛い・愛嬌・甘さ | ①まるっと可愛い ②あざと可愛い ③ポップ ④スイート |
| 4 | **エレガント** | Elegant | 上品・華やか・洗練 | ①クラシカル ②華やか ③フェミニン ④グレイスフル |
| 5 | **クール** | Cool | 知的・シャープ・モード | ①モード ②知的クール ③ミニマル ④アーバン |

### 3-3. 2軸マッピング（AI検出の基礎座標）

```
                    フレッシュ（Fresh）
                         |
            フレッシュ    |    キュート
            (2)          |    (3)
                         |
  クール ─────────────────+───────────────── ウォーム
  (5)                    |                   (8)
                         |
            ミステリアス  |    ナチュラル
            (7)          |    (1)

                    リッチ（Rich/Mature）

  ※ エレガント(4)は右上〜右寄り
  ※ ワイルド(6)は左下〜左寄り
```

### 3-4. 各タグに対応するAI検出特徴マッピング

| Vibe Type | AI検出する顔の特徴（技術的パラメータ） |
|-----------|--------------------------------------|
| **ナチュラル** | 曲線的輪郭、中間的目の大きさ、柔らかい表情筋の状態、中〜低コントラスト顔 |
| **フレッシュ** | 直線的要素あり、明るい肌トーン（※色味ではなく明度）、すっきりした目元、左右対称性が高い |
| **キュート** | 曲線的輪郭、大きめの目（顔比率）、短い顔の縦幅比率、丸みのあるパーツ |
| **エレガント** | 曲線的＋パーツ大きめ、顔の縦幅比率が長め、目と眉の間隔、全体バランスの整い |
| **クール** | 直線的輪郭、シャープな顎角度、切れ長の目（横幅比率）、高い鼻筋ライン |
| **ワイルド** | 直線的＋パーツ大きめ、太い眉、顎の角度、彫りの深さ（陰影） |
| **ミステリアス** | 直線〜中間、低い眉位置、目の奥行き感（陰影深め）、非対称的魅力 |
| **ウォーム** | 曲線的、穏やかな目元の角度、口角の上がり具合、全体的な丸み |

**重要**: AIは「顔の幾何学的特徴」から「印象カテゴリ」へのマッピングを行う。直接的な容姿の良し悪しは判定しない。

## 4. 相性行列（マッチングマトリクス）

### 4-1. マッチング2タイプの定義

| マッチタイプ | 定義 | 心理学的根拠 |
|-------------|------|-------------|
| **統一感マッチ** | 同系統のVibeで「しっくりくる」ペア | 類似性の法則（Similarity-Attraction Effect）：似た特徴を持つ人同士は好感度が高い |
| **補完マッチ** | 対比的なVibeで「お互いが映える」ペア | 補完性仮説（Complementarity Hypothesis）：異なる特徴が互いを引き立てる |

### 4-2. 統一感マッチ 相性行列

相性スコア: ◎=最高 / ○=良い / △=普通 / -=低い

**統一感マッチのロジック**:
- 2軸マップ上で「近い位置」にあるタイプ同士が高相性
- 同じタイプ同士は常に◎（最も統一感がある）
- ソフト系同士（ナチュラル/キュート/ウォーム）は相互に○以上
- シャープ系同士（クール/ミステリアス）は相互に◎

### 4-4. 補完マッチ 注目ペア（対比で映える組み合わせ）

| ペア | 相性根拠 | ファッション/心理学知見 |
|------|---------|----------------------|
| **キュート × クール** | 甘辛ミックス。可愛さとシャープさの対比が映える | ファッションの「甘辛コーデ」理論。対比が視覚的インパクトを生む |
| **ナチュラル × ミステリアス** | 素朴さと神秘性。安心感×ドキドキの組み合わせ | 心理学の「最適覚醒理論」：適度な新奇性が魅力を高める |
| **フレッシュ × ワイルド** | 爽やかさと力強さ。清涼感×ダイナミクスの補完 | 色彩心理学のコントラスト効果。寒色×暖色の補色関係に近い |
| **フレッシュ × エレガント** | 若々しさと洗練。カジュアル×フォーマルの対比 | スタイリング理論の「ドレスダウン/アップ」効果 |
| **クール × ウォーム** | 冷たさと温かさ。知的×包容力の典型的補完 | 対人魅力研究：温かさと能力の2次元モデル（Fiske et al.） |
| **ミステリアス × ウォーム** | 深み×安心感。引き込まれる魅力×癒しの組み合わせ | 「ギャップ萌え」現象。予想外の組み合わせが記憶に残る |
| **キュート × ワイルド** | 可愛い×力強い。守りたい×守ってくれるの双方向 | 進化心理学的な相互補完。異なる資質の組み合わせ |
| **ナチュラル × エレガント** | 素朴さ×華やかさ。日常×非日常の対比 | ゲシュタルト心理学の図と地の関係。片方が他方を際立たせる |

### 4-5. マッチングアルゴリズムへの実装指針

```
マッチスコア計算:

  total_score = α × unity_score + β × complement_score + γ × preference_score

  α = 統一感重み（デフォルト 0.4）
  β = 補完重み（デフォルト 0.3）
  γ = ユーザー好み重み（デフォルト 0.3）

  ※ ユーザーが「統一感重視」「補完重視」を選べるUIにする
  ※ preference_scoreはユーザーのいいね履歴から学習
```

## 5. 実装上の注意・倫理ガイドライン

### 5-1. UI/UXでの配慮

| 項目 | ガイドライン |
|------|------------|
| **タグの表示** | 全タグを同じ大きさ・色で表示。特定タグを目立たせない |
| **自己編集権** | ユーザーはAI推定タグをいつでも変更・削除・追加可能 |
| **複数タグ** | 1人につきメインVibe 1 + サブVibe 2-3。多面性を表現 |
| **説明文** | 各タグに「あなたの魅力」としてポジティブな説明を表示 |
| **比較しない** | 「○○顔の人は△%」等の統計表示はしない（序列化防止） |
| **オプトアウト** | AI推定を使わず、完全手動選択も可能にする |

### 5-2. やってはいけないこと

1. **顔のスコアリング**: 「美人度80点」等は絶対にNG
2. **タグの序列化**: 人気タグランキング等の表示禁止
3. **身体的特徴の直接記述**: 「目が大きい」等はバックエンドのみ。ユーザーには見せない
4. **肌色・人種推定**: 技術的に可能でも絶対に実装しない
5. **性別固定タグ**: 「男性向けタグ」「女性向けタグ」の分離禁止
6. **ネガティブタグ**: 「地味」「薄い」等のタグは存在させない

### 5-3. 法的リスク対応

| リスク | 対応 |
|--------|------|
| 個人情報保護法 | 顔画像の取得・解析に明示的同意を取得。目的外利用禁止 |
| 差別助長 | タグ体系の定期的な外部レビュー。差別表現チェック委員会の設置 |
| プライバシー | 顔解析はデバイス上（オンデバイスAI）で完結させ、サーバーに顔画像を送らない選択肢 |
| 炎上対応 | プレスリリース・FAQ・想定Q&Aの事前準備 |
