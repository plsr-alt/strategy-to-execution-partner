# 納品物チェックルール

## 対象ドキュメント
設定書・報告書・手順書・仕様書等、顧客・社外への納品物全般。

---

## チェック項目一覧

### 1. 命名規則
- [ ] ファイル名が規則に準拠しているか
- [ ] 版番号がファイル名に含まれているか（例: `_v1.0`）
- NG時：正しいファイル名を提示する

### 2. 文書番号・採番
- [ ] 文書番号が正しく採番されているか
- [ ] 版番号がファイル名と文書内ヘッダで一致しているか
- NG時：正しい採番を提示する

### 3. フォーマット・拡張子・PDF変換
- [ ] 指定テンプレートを使用しているか
- [ ] 拡張子が要件に合致しているか（.docx / .xlsx / .pdf 等）
- [ ] 納品時のPDF変換が必要な場合、変換済みか

### 4. 改訂履歴
- [ ] 版番号・日付・変更内容・作成者が表に記載されているか
- [ ] 版番号が前版から正しくインクリメントされているか（1.0→1.1 等）

### 5. 前版との差分確認
- [ ] 意図しない削除・記述漏れがないか
- [ ] 修正箇所が正しく反映されているか（修正依頼内容との突合）

### 6. 過去指摘事項の解消確認
- [ ] 前回レビューの指摘事項がすべて対応済みか
- [ ] 未対応の指摘がある場合、理由が明記されているか

### 7. 提出先・宛先情報
- [ ] 担当者名・会社名・部署名の誤記がないか
- [ ] 提出先の要件（書式・送付方法・部数）を満たしているか

### 8. 機密レベル・取扱注意の記載
- [ ] 機密レベルの記載が必要な場合、正しく記載されているか（例: 社外秘・取扱注意）
- [ ] 不要な内部情報が含まれていないか

### 9. 誤字脱字・表記ゆれ
- [ ] 明らかな誤字脱字がないか
- [ ] 社名・製品名・専門用語の表記が文書内で統一されているか（例: サーバ vs サーバー）

---

## 判定基準

| 判定 | 条件 |
|------|------|
| **OK** | 全該当チェック項目がクリア |
| **要確認** | 判断が必要な曖昧な項目あり（ユーザーに確認） |
| **NG** | 1件以上の明確な違反あり |

---

## 是正手順（NGが出た場合）

1. NGファイルを `old_<元ファイル名>` にリネームして同フォルダに退避
2. 元ファイル（または新規ファイル）を修正
3. 再チェックを実施（本ルール最初から）
4. OK確認後、`old_` ファイルを `99_ARCHIVE/` へ移動
