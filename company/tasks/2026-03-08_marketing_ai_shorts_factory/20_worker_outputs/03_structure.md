## 全体構成

AI Shorts Factory の市場投入・運用・最適化を支える組織・フロー・KPI体系を、ブランド定義→ターゲット→ファネル→KPI→組織→メディア別戦略→コンテンツフロー→実行計画→レビュー体制の9セクションで統合。本設計は既存 YouTube パイプライン資産を活用しながら、YouTube Shorts / Instagram Reels / TikTok への3プラットフォーム同時展開を支える。

## セクション1：ブランド定義

### 1.1 コンセプト
- **タイトル**: AI Shorts Factory
- **キャッチコピー**: 「無料 AI アートで毎日の発見を」
- **本質**: FLUX.2[klein] + Ken Burns + Groq による 完全ゼロコスト自動生成エンジンで、ショート動画を毎日3本量産し、3プラットフォームに同時展開

### 1.2 事業目的
1. **既存 YouTube パイプライン資産の活用** — moviepy / Groq / EC2環境を転用
2. **リーチ最大化** — YouTube Shorts 発見度 + Instagram 保存度 + TikTok エンゲージメント の加重レバレッジ
3. **収入源多角化** — YouTube AdSense（月¥30-50K）+ Instagram バッジ + TikTok Creator Fund（月計¥70K推定 / 12ヶ月目）
4. **Rena キャラ育成への接続** — ショートフォーマットで認知度構築 → キャラ販売・デジタルグッズへ流入

### 1.3 差別化ポイント
| ポイント | 実装内容 |
|---------|--------|
| **コスト構造** | ¥0/月実現（EC2共用+GCS¥100のみ）。Midjourney/Runway 等の有料モデルを除外 |
| **テンプレ多様化** | 単一テンプレ駆動を脱却。4テンプレ → 7テンプレへ段階拡張 |
| **テーマニッチ化** | 「金融×AI」「教育×AI」など複合テーマで検索ニッチ獲得 |
| **高頻度投稿信号** | 3本/日で「アクティブ」判定 → アルゴリズム有利（競合は平均 1-2本/日） |
| **動画品質** | Ken Burns アニメーション + Wan 2.1 フォールバック で「静止画→動画」品質感を維持 |

### 1.4 NG トーン・表現
- **実在人物の顔生成禁止** — プロンプトに「人物」「顔」「実在」含めない
- **リンク誘導スパム** — 月3本以下に抑制（アルゴリズム低評価回避）
- **著作権侵害** — Pixabay / YouTube Audio Library のみ使用（Content ID マッチなし）
- **金融免責事項省略** — 金融テーマには必ず「投資助言ではありません」表記
- **AI明記省略** — キャプション末尾に #AIアート / #生成AI タグ必須

### 1.5 プロジェクトステータス
- **意思決定**: 仕様確定（SPEC.md v1.1）
- **実装フェーズ**: Phase 0（セットアップ） → Phase 1（パイプライン） → Phase 2（テスト投稿） → Phase 3（本運用）
- **GO/NO-GO判定**: Phase 2 終了時（3/25-4/7）に数値ベース判定

## セクション3：ファネル設計

### 3.1 4段階マーケティングファネル

#### Stage 1: Awareness（認知・視聴）
```
投稿方法: YouTube Shorts / Instagram Reels / TikTok へ毎日2-3本投稿
到達媒体:
  - YouTube Shorts: 「発見」タブ（推奨度スコア順）
  - Instagram Reels: フィード + Explore Tab（ハッシュタグトリガー）
  - TikTok: For You Page (FYP)（完全視聴率+エンゲージメント加重）

成功指標:
  - YouTube: 新規視聴者数 ≥ 50/本
  - Instagram: リーチ ≥ 500/本
  - TikTok: インプレッション ≥ 1,000/本
```

#### Stage 2: Interest（興味深化・フォロー）
```
トリガー: 「この人のコンテンツをもっと見たい」という欲求発生
CTAメッセージ:
  - キャプション: 「#AIアート #FLUX で毎日更新」
  - テキストオーバーレイ: 「フォローして次の作品をチェック」
  - ハッシュタグ: #AIアート #生成AI #FLUX2 #AIイラスト

成功指標:
  - YouTube: フォロー率 ≥ 3-5%（完全視聴者から）
  - Instagram: フォロー率 ≥ 2-3%
  - TikTok: フォロー率 ≥ 1-2%（低い傾向。シェア優先）
```

#### Stage 3: Consideration（エンゲージメント・検討）
```
ユーザー行動: 保存 → コメント → プロフィール訪問
プロフィール内容最適化:
  - 説明欄: 「毎日AIアートを投稿。YouTubeチャネル登録で長尺版も」
  - リンク: YouTube チャネル（MAX 1-2個） / BOOTH イラスト販売

エンゲージメントCTA:
  - キャプション末尾: 「この絵柄好きですか？コメントで教えてください」
  - テキストオーバーレイ: 「保存して後で見返してね」

成功指標:
  - YouTube: コメント率 ≥ 1%（低めが正常）
  - Instagram: 保存率 ≥ 3-5%（保存を重視）
  - TikTok: シェア率 ≥ 5-10%（最重要指標）
```

#### Stage 4: Conversion（流入・販売）
```
流入先:
  - YouTube Shorts 説明欄 → YouTube チャネル登録（目標: 月 +500登録）
  - Instagram Reels プロフィール → BOOTH / FANBOX （目標: 月 ¥5,000-15,000）
  - TikTok プロフィール → YouTube / BOOTH（間接流入）

Monetization:
  - YouTube Partner Program（AdSense）
  - Instagram Reels Bonus Program
  - TikTok Creator Fund（10,000フォロワー後）
  - BOOTH デジタル販売（イラスト素材・PromptシートPDF）

成功指標:
  - YouTube Shorts からの YouTube チャネル流入 ≥ 20/日
  - BOOTH 売上 ≥ ¥3,000/月（初期）→ ¥15,000/月（6ヶ月後）
  - YouTube AdSense 推定月収 ≥ ¥5,000/月（3ヶ月後）
```

### 4.4 撤退基準（予測）

**撤退判定タイミング**: Phase 2 末（4月初旬）

**撤退トリガー条件**:
```
[即撤退]
1. AVG_VIEWS < 200/本 かつ 改善方法が特定できない
2. エラー発生率 > 20% + 技術的再現が困難
3. 規約違反が頻発（3件/週以上）

[要評価・改善後判定]
4. エンゲージメント率 < 1% だが AI プロンプト最適化で回復可能な場合 → Phase 3 へ延期
5. API レート制限が頻発 → キューイング・バッチ処理で対応可能か評価
```

**撤退時アクション**:
```
- プロジェクト凍結（新投稿停止）
- 既成投稿は削除しない（既存ユーザーへの価値は継続）
- スクリプト・パイプライン資産は `99_ARCHIVE/` へ退避
- 学習ドキュメント（KNOWLEDGE.md）に失敗要因を記録
- Rena LoRA や デジタル販売 など他PJへのリソース振り分け検討
```

### 5.3 AI層（4システム）

#### ロール3: 台本・プロンプト生成エンジン
| 項目 | 内容 |
|------|------|
| **システム名** | Groq LLM (無料API) |
| **目的** | 毎日のテーマ × 動画台本 × プロンプト を自動生成 |
| **API** | Groq API（無料枠） / モデル: mixtral-8x7b |
| **インプット** | テーマリスト（金融/AI/教育） + 日付 + 前週KPI |
| **アウトプット** | JSON形式：台本 / プロンプト / BGMテンポ / キャプション |
| **制約** | 禁止キーワード（人物/顔/実在人物）をフィルタリング |
| **1日処理量** | 3本/日 （90秒/本） |
| **エラーハンドリ** | 生成失敗時は テンプレ台本フォールバック |
| **成果物** | `daily_scripts.json` (タイムスタンプ付き) |

#### ロール4: 画像生成エンジン
| 項目 | 内容 |
|------|------|
| **システム名** | FLUX.2 [klein] (4B, Apache 2.0) |
| **目的** | Groq 台本 → AI画像 3-5枚 を高速生成 |
| **環境** | ComfyUI on EC2 t4g.medium (VRAM 8GB) |
| **実行時間** | 0.5秒/枚 × 4枚 = 2秒/本（含シード固定） |
| **フォールバック** | VRAM不足時 → Wan 2.1 / AnimateDiff |
| **品質チェック** | 人物顔検出→自動削除 / 暴力的表現フィルタ |
| **1日処理量** | 3本 × 4枚 = 12枚/日 |
| **成果物** | PNG 1024x1024 フォルダ (date_id 付き) |

#### ロール5: 動画化・アニメーションエンジン
| 項目 | 内容 |
|------|------|
| **システム名** | Ken Burns + Wan 2.1 / FFmpeg |
| **目的** | 静止画 → 短尺動画（15-60秒）への変換 |
| **処理フロー** | FLUX画像 (4枚) → Ken Burns ズーム/パン → Wan 2.1（オプション）で動画化 |
| **出力形式** | MP4 (H.264, 1080p, 30fps) |
| **実行時間** | 15秒/本（含エンコード） |
| **フォールバック** | Wan 2.1 失敗時 → AnimateDiff / CogVideoX |
| **1日処理量** | 3本/日 |
| **成果物** | `outputs/raw_videos/` MP4ファイル |

#### ロール6: 編集・合成エンジン
| 項目 | 内容 |
|------|------|

```
[投稿戦略]

[エンゲージメント施策]

- コメント促進: 「この絵柄どう？」などの選択肢提示
```

### 9.3 全否定部隊（Devil's Advocate）

全否定部隊は、プロジェクト継続の判断を厳しく検証する機能。月1回、以下を検査:

| 検査項目 | 全否定視点 | 判定基準 |
|---------|----------|--------|
| **テーマ妥当性** | 「この4つテーマで本当に差別化できるのか？競合との区別は？」 | テーマ別 AVG_VIEWS で 競合比較表を作成。明確な優位性あるか |
| **技術リスク** | 「FLUX.2[klein] + Ken Burns で本当に YouTube 基準を満たす品質が出ている？」 | 実投稿 10本の 視聴者コメント / 低評価率で判定。基準未満なら改造必須 |
| **規約リスク** | 「AI顔生成フィルタで本当に顔が出ていないのか？Content ID マッチはゼロなのか？」 | Content ID 検査実施 / 外部ツール（TubeRanker等）で 顔検出テスト |
| **経営効率** | 「月90本投稿で¥3,000-8,000月収は見合うか？1人運用の人件費考えると ROI 悪い？」 | RPM計算：¥8,000 / (90本×平均500ビュー) = 時給換算。基準値以下なら撤退検討 |
| **スケーラビリティ** | 「TikTok の1日1本制限や YouTube の月100本推奨ラインで本当に月90本が最適？」 | プラットフォーム別上限値の再確認。違反のリスク定量化 |
| **代替案検討** | 「この PJ の代わりに Rena LoRA や デジタル販売 に資源を振った方が ROI 高くないか？」 | 他PJ の期待値（RPM計算）と比較。PJ 継続・中止・ハイブリッドを検討 |

**全否定部隊の権限**:
- 提案段階での重大リスク指摘 → CEO/本人に報告
- Phase 2 NO-GO の可能性を事前指摘 → 改善方策立案
- 撤退判断の最後のセーフティネット（「実はこのPJ、やめた方が良いかも」という意見表明）
