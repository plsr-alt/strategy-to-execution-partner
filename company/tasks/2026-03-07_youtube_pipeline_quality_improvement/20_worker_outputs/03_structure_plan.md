### 3. 改善機能仕様（優先度1：音声）
- **内容**: gTTS → VOICEVOX への置き換え実装
  - **現行コード** (Lines 311-343):
    ```python
    from gtts import gTTS
    tts = gTTS(text=full_text, lang='ja', slow=False)
    ```
  - **改善案**:
    - ライブラリ: VOICEVOX Python API (`python-voicevox` or HTTP REST API)
    - 関数署名: `def generate_voiceover_voicevox(script: str, speaker_id: int = 3, speed: float = 1.2) → str`
    - 実装内容:
      - 文単位で分割（句点「。」で分割）
      - 各文の末尾に自動ポーズ挿入（500ms）
      - VOICEVOX サーバーに HTTP POST で音声生成リクエスト
      - MP3 に変換して保存（VOICEVOX はWAVで返す場合があるため）
    - テスト: `VOICEVOX_URL` 環境変数確認、音声ファイル生成確認
  - **優先度**: 1位（高インパクト・低難度）
  - **実装期間**: 1-2週

### 4. 改善機能仕様（優先度1：映像）
- **内容**: 映像素材の改善（テロップ・図解・キャラクター統合）
  - **現行コード** (Lines 400-468: `generate_infographics()`):
    - matplotlib で単純な図解スライドを生成しているが、テロップ・キャラクターなし
  - **改善案**:
    - テロップレイヤーの追加:
      - 関数: `def add_telop_to_clips(footage: list, script: str) → list`
      - 実装: moviepy + PIL で、台本の各セクションに対応したテロップを動的生成
      - 配置: 画面下部、ゴシック体50pt、背景: 黒半透明、文字: 白
    - 図解の強化:
      - 関数: `def generate_enhanced_infographics(theme: dict, num_slides: int = 3) → list`
      - 実装内容:
        - スライド1: タイトル + キーワード3個
        - スライド2: 3ポイント比較（カラーボックス＋矢印）
        - スライド3: データグラフ（仮想データ数値化）
      - ライブラリ: matplotlib + seaborn（スタイル向上）、Pillow（テキスト配置）
    - キャラクター配置:
      - 関数: `def add_character_overlay(footage: list, character_sprite_path: str) → list`
      - 実装: 画面左右にキャラクター画像を透過合成
      - 仕様: PNG + alpha チャンネルで透過処理
  - **優先度**: 1位（高インパクト・低難度）
  - **実装期間**: 2-3週

### 5. 改善機能仕様（優先度2：動画尺統一）
- **内容**: 動画尺を8-15分 → 18-25分（目標20分）に統一
  - **現行コード** (Lines 47-48):

  - **優先度**: 2位（高インパクト・低難度）
