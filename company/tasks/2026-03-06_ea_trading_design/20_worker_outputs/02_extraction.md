## 1. EAコンセプト候補素材抽出：制約条件下での戦略選定

### 制約条件の確認
- **月次利益目標**: 3～5%（年利36～60%に相当）
- **MaxDD（最大ドローダウン）**: 20%以内
- **ナンピン**: NG（国内規制＋ブローカー約款。21行目参照）
- **マーチンゲール**: 黙認状態だが避けるべき（22行目参照）
- **出口**: 固定ロット + ATRベース SL設定が代替手法（21行目、14行目参照）

---

### 候補1：トレンドフォロー型（EMA+ADXクロス）

#### 概要
EMA短期・中期クロス + ADX強度フィルター + ATR SL設定

#### パラメータ基準（11行目、12行目より）
- **EMA**: 短期5/9日 × 中期20/25日クロス（1H足推奨）
- **ADX**: 14期間・20以上でエントリー許可（トレンド形成段階以上）
- **SL**: ATR×3倍をベース（14行目）
- **ロット**: リスク率2%固定（口座資金の2% ÷ SL pips ÷ pip当たり価値）

#### 強み
- **高信頼度パラメータ**: 業界標準EMA/ADX組み合わせで最適化リスク低い
- **スプレッド耐性**: トレンド保有期間長い→スプレッド0.2銭での損益影響小
- **サンプルサイズ稼ぎやすい**: 1時間足・複数通貨ペア対応で取引数多数確保

#### 弱み
- **勝率低**: 40～50%が現実的（4行目より）→心理的ストレス。マイナス連敗でロジック疑い強まる
- **Max DD大**: トレンド反転待つため15～25%（4行目より）→20%制約で余裕わずか
- **トレンドレス相場で機能停止**: ADXフィルター機能してもダマシ多数

#### リスク
- **バックテスト≠リアル乖離**: スリッページ実際+3～10pips（9行目）→勝率が実績値より3～5%低下可能性高い
- **スプレッド変動時**: 0.2銭基準で計算したSLが拡大時に20% MaxDD超過リスク
- **過剰最適化陥穽**: 複数パラメータ（EMA周期・ADX強度・TP倍数）の過度な最適化でフォワード崩壊80%超（19行目）

#### 開発難度
- **低** - EMA・ADXはMQL4標準ライブラリ・オンライン記事豊富。実装リスク小

---

### 候補2：レンジ型（ボリンジャーバンド+RSI逆張り）

#### 概要
±2σタッチ + RSI 30/70フィルター + 上下線内での反発トレード

#### パラメータ基準（13行目より）
- **ボリバン**: 20期間・±2σ（デフォルト）
- **RSI**: 14期間・フィルター閾値30/70
- **SL**: ±2σ外側5～10pips
- **ロット**: リスク率2%固定

#### 強み
- **高勝率**: 60%以上が現実的（4行目より）→心理的安定性高い・ドローダウン心理的耐性向上
- **MaxDD小**: レンジ相場得意で反発待つため短期保有・リスクリワード小（4行目）→DD 10～15%で済む可能性
- **移動平均より粘り強い**: レンジ相場時間が長い（トレンド占有率30～40%）→取引機会多い

#### 弱み
- **トレンド相場で大損**: ブレイク時にボリバン上下線突き抜け→SL狩られ連敗（ダマシ多数）
- **リスクリワード悪化**: 反発トレードで利幅小・損失大になる局面多い
- **TP設定の恣意性**: ボリバン内での目標位置が明確でない→パラメータ最適化陥穽高い

#### リスク
- **RSIダマシ**: 相場が強いトレンド中もRSI70以上で「売られすぎ」と誤判定→逆張りで焼ける
- **スプレッド拡大でSL発動多発**: 変動スプレッド時にボリバン内での小さなSLが頻繁に刺さるリスク
- **過剰最適化：RSI・BB周期の過度な調整で局所最適化**

#### 開発難度
- **低～中** - ボリバン・RSI標準実装。ただしレンジ判定ロジック（相場局面判別）がMQL4で複雑

---

### 候補3：モメンタム型（ATRブレイク＋EMA方向性フィルター）

#### 概要
ATR計算して直近N足の高値/安値ブレイク時にエントリー。中期EMAで方向フィルター。

#### パラメータ基準（14行目より）
- **ATR**: 14期間・ブレイク幅決定用
- **ブレイク参照足**: 20足～50足直近高値/安値
- **EMA**: 中期20/25日で上向き/下向き確認
- **ロット**: リスク率2%固定（ATR×2倍を基本SL）

#### 強み
- **トレンド初動キャッチ**: レンジ離脱時の加速局面を捉える→利益伸ばしやすい
- **ブレイク明確**: エントリー/SLロジック単純・パラメータ少ない→過剰最適化リスク低い
- **ボラティリティ調応**: ATRで相場状況に応じたSL自動調整→固定値より柔軟

#### 弱み
- **ダマシ多い**: ブレイク失敗（戻し）が30～40%確度→連敗リスク
- **方向性フィルター不足**: EMA単一指標では弱いトレンド区別が甘い
- **M足相場での機能不全**: 下位足ブレイクが上位足で抵抗になる場合→シグナル無効化

#### リスク
- **スリッページによるブレイク無効化**: ブレイク確定がスリッページで吸収される可能性（9行目）
- **TP設定の不確実性**: ブレイク後の利益伸ばし方が曖昧→パラメータ多重化陥穽
- **短期足への過度な依存**: 1H足以下での実装だとスプレッド・スリッページ影響大

#### 開発難度
- **中** - ATRブレイク検出ロジック、EMAフィルター統合の正確なコーディング要

| 項目 | トレンドフォロー型 | レンジ型 | モメンタム型 |
|------|------------------|---------|------------|
| **強み度** | 中（信頼度・サンプル数） | 高（勝率・DD小） | 中（初動キャッチ） |

| **制約適合性** | ○（MaxDD20%ギリギリ） | ◎（MaxDD10~15%で余裕） | ○（ダマシ制御で20%以内） |
| **推奨優先度** | 2番目（検証難度低） | **1番目（DD制約クリア率高）** | 3番目（ロジック複雑） |

## 2. MVP仕様の論点整理：最小検証単位の仕様定義

### MVP制約条件
- **1戦略・1通貨ペア**: レンジ型（候補2）推奨（DD制約クリア率最高）
- **通貨選定根拠**: 国内ブローカー標準スワップ・スプレッド考慮→**ドル円（USD/JPY）**
  - スプレッド：0.2銭以下（国内標準。3行目より）
  - スワップ：トレンド戦略向け標準通貨（25行目より）
  - ボラティリティ：FXエントリー層向けスタンダード（流動性充分）

### MVP必須パラメータセット

#### 1. インジケーター最小構成
| # | インジケーター | パラメータ | 根拠 |
|---|--------------|----------|------|
| 1 | ボリンジャーバンド | 期間20・±2σ | 13行目「標準設定」 |
| 2 | RSI | 14期間・閾値30/70 | 13行目「フィルター基準」 |
| 3 | ATR | 14期間 | 14行目「SL設定用」 |

#### 2. エントリー・ポジション管理ロジック
| # | 要素 | 仕様 | 数値根拠 |
|---|-----|------|---------|
| 1 | エントリー条件 | ボリバン±2σタッチ + RSI 30以下（売り）or 70以上（買い） | 13行目より |
| 2 | フィルター | - | RSI単独エントリー禁止。ボリバン確認必須 |
| 3 | SL（損切り） | ATR × 2倍 | 14行目「中期足で1%前後」を目安に設定 |
| 4 | TP（利確） | ボリバン中心線（20MA）到達時 | 簡潔性・検証容易性 |
| 5 | ロット計算 | 固定リスク2%方式：Lot = (口座残高 × 0.02) ÷ (SL pips × 100) | 15行目より |
| 6 | ポジション数上限 | 最大1ポジション（同方向複数禁止） | ナンピン禁止対応（21行目） |

#### 3. バックテスト検証に必要なデータ期間・Tick品質

##### データ期間
- **最小期間**: 1年間（12ヶ月）
  - **根拠**: 相場環境4～6シーズン（トレンド・レンジ・ボラティリティ上昇・低下）を最低1回網羅（6行目「サンプル数2000以上」の目安）
  - **実装検証**: 直近2～3年がベスト（過去のパラメータ最適化でのフォワード検証も可能）
- **最大期間**: 5年間以上推奨（相場環境変化対応性検証）

##### Tick品質・モデル選定
| 項目 | 仕様 | 根拠・トレードオフ |
|------|------|------------------|
| **粗検フェーズ** | Open prices only（MT5バックテスト） | 17行目。処理時間短・概略精度80%→3～4日内に実施 |
| **精細検証フェーズ** | Every tick based on real ticks（Tick Data Suite推奨） | 10行目「99.9%精度」。ただし処理時間48～72h要。MVPでは必須 |
| **スプレッド設定** | 固定0.2銭（国内標準。変動スプレッド考慮で+0.1銭）= 0.3銭設定 | 3行目・9行目より。バックテスト乖離を20～30%見込む余裕設定 |
| **スリッページ設定** | 実際+5pips固定 | 9行目「実際+3～10pips」の中位値。バックテストで遠回し的に組込み |

##### 最小サンプル数（売買回数）
- **目標**: 取引数 **2,000回以上**
  - **根拠**: 6行目「2000回以上で過剰最適化が施されていないと考えられる」
  - **レンジ型での達成難度**: 1年間・1通貨ペアでも1H足なら2000回超過は容易（平均3～5回/日 × 250営業日）

#### 4. MVP検証フロー

1. **粗検**: Open prices only で3日以内に実施
   - 結果確認ポイント: 勝率 > 50% / Profit Factor > 1.3

2. **精細検証**: Tick Data Suiteで Every tick実施（土日含め5日）
   - 期間: 直近12ヶ月
   - 期待結果: 粗検比で-20～30%の成績低下を許容（バックテスト≠リアル乖離考慮）

3. **ウォークフォワード分析**: IS/OOS 70:30分割
   - 計3～4回の窓を移動させて再検証
   - OOS成績が IS の-40%以内なら合格（8行目参照）

## 3. V1仕様の論点整理：MVPからのスケールアップ

### V1での追加要素・モジュール化の必要性

#### 1. 複数通貨ペア対応

**必要性根拠**:
- MVPの1ペア（ドル円）でのサンプルサイズ2000回達成には1年要する
- V1で複数ペア対応（3～5通貨）により、同期間内で6000～10000回取引確保 → 統計信頼度3～5倍向上
- スプレッド・スワップ特性が異なるペア間での戦略ロバスト性検証

**追加モジュール**:
- マルチペア初期化: OnInit()で複数ペア登録・インジケーター並列計算
- ペアごとの独立ロット計算: 各ペア別口座比率管理
- 通貨間相関フィルター: 同時エントリー抑制ロジック（過度なリスク集中回避）

#### 2. 相場環境識別フィルター追加

**必要性根拠**:
- レンジ型戦略がトレンド相場で大損する問題（候補2「弱み」より）
- ADXベースのトレンド/レンジ判別フィルター追加でダマシ低減
- 参考: 候補1で既出のADX 20以上=トレンド判定を逆利用 → レンジ時のみエントリー許可

**追加モジュール**:
```
相場局面判別ロジック:
- ADX < 20 → レンジ相場判定（ボリバン+RSI有効）
- ADX >= 20, 40未満 → トレンド形成段階（警戒モード）
- ADX >= 40 → 強トレンド（全エントリー停止）
→ 各局面でロット調整: レンジ100% / 形成段階50% / 強トレンド0%
```

根拠: 12行目「ADX強度判定」パラメータ仕様

#### 3. 動的SL/TP調整ロジック

**必要性根拠**:
- MVPの固定SL（ATR×2）では、市場ボラティリティ変化に非対応
- 例: コロナショック時（VIX 60+）では ATR×2では不足 / 静寂時はATR×2では広すぎる
- リアル運用での口座一貫性向上

**追加モジュール**:
```
動的SL計算:
SL_pips = ATR × (基本倍数 × ボラティリティスコア)
where ボラティリティスコア = 直近ATR / 20日平均ATR
→ 相場静寂時: SL縮小 / ボラティリティ上昇時: SL拡大
```

根拠: 14行目「ATR×3倍」での可変性記述

#### 4. ウォークフォワード自動検証フレームワーク

**必要性根拠**:
- MVPは静的バックテスト（全期間一括最適化）
- V1では IS/OOS分割テストを自動化 → パラメータドリフト検出
- 定期的（月1回）な自動再検証で陳腐化の早期発見

**追加モジュール**:
```
検証フレームワーク:
1. データセット分割: 過去30ヶ月を10週間単位で分割
2. ローリング最適化: 各週でIS 70%, OOS 30%設定
3. パフォーマンス比較: OOS結果 vs IS の-40%超過警告
4. アラート: 継続検証で IS-OOS乖離が拡大傾向なら警告
```

根拠: 8行目「ウォークフォワード分析の標準的IS/OOS比率」

#### 5. ログ・統計・可視化モジュール

**必要性根拠**:
- MVPではターミナルログのみ
- V1では月次報告書自動生成・チャート分析（最大ドローダウン時期の可視化）
- リアル運用開始時の外部報告・透明性確保

**追加モジュール**:
- CSV出力: 全取引履歴（エントリー時刻・価格・SL・TP・クローズ価格・P&L）
- グラフ化: 月次利益推移・キュムリティブP&L・ドローダウン曲線
- サマリー: Profit Factor / Max DD / 勝率 / シャープレシオ / ソルティノレシオ

#### 6. リスク管理強化：ストップロス後の再エントリー禁止タイマー

**必要性根拠**:
- SL狩られた直後の感情的な再エントリーでロット増加・追加損失
- MVP では存在しないが、リアル運用で心理的なドローダウンが多い
- X分以内の連続エントリー禁止フラグで機械的に制御

**追加モジュール**:
```
再エントリータイマー:
SL発火後 → 30分～2時間のクールダウンタイマー設定
→ 期間内のエントリーシグナル無視
```

## 4. GO/NO-GO基準の数値根拠抽出

### 4.1 公開EA実績ベンチマーク（信頼度中～高）

#### 参考データソース
- **MQL5 Market最上位EA**: 20ヶ月以上のリアルトレード記録公開（23行目）
- **業界統計**: プロフィットファクター解説・FX利回り分析・EA職人講座（複数出典）

#### GO基準：現実的な成績期待値（中央値 & 上位25%）

| 指標 | 中央値 | 上位25%値 | 根拠・注釈 |
|------|-------|---------|----------|
| **月利（%）** | 3～5% | 8～12% | 24行目「月利3～5%は理論的に可能だが安定成績目安」。上位25%は月利10%超はハイリスク裏返し |
| **年間勝率（%）** | 50～55% | 65～70% | 4行目「トレンド40～50%」「レンジ60%以上」の加重平均。スリッページ・スプレッド反映済み |
| **Profit Factor** | 1.5～1.8 | 2.0～2.5 | 5行目「適正1.3～2.0」。上位は2.0超 |
| **Max DD（%）** | 12～18% | 8～12% | 4行目「トレンド15～25%」「レンジ10～15%」の実績。20%制約に対してGO基準は余裕持たせた |
| **シャープレシオ** | 0.8～1.2 | 1.5～2.5 | 年間リターン÷標準偏差。月利5%で計算した場合の推定値 |
| **連敗回数（最大）** | 5～8回 | 3～5回 | 勝率50%程度での理論値。19行目「5以下は危険シグナル」の反対解釈 |

#### NO-GO基準：過剰最適化＆聖杯EAの危険シグナル（複合判定）

以下の複数項目に該当する場合、フォワード崩壊80%超の可能性あり（7行目より）→ **NO-GO** 判定：

| # | 危険シグナル | 閾値 | 判定ロジック | 信頼度 |
|---|-------------|------|-----------|--------|
| 1 | **PF（過剰値）** | PF ≥ 3.0 | 取引数が2000以上あっても3.0超は要注意。理由：取引環境依存度高、パラメータドリフト可能性 | 高 |
| 2 | **勝率（過剰高）** | 勝率 ≥ 85% | 85%超は逆張り過剰最適化またはナンピン・マーチンゲール違法トレード疑い | 高 |
| 3 | **サンプル数（不足）** | 取引数 < 500 | 6行目明示。500回以下は統計的信頼度0。必ずNO-GO | 高 |
| 4 | **連敗数（過剰少）** | 最大連敗 ≤ 5 | 19行目「5以下は危険」。確率論上、50%勝率で最大連敗3～4が期待値。それ以下は環境特異化 | 中 |
| 5 | **Max DD（小さすぎ）** | Max DD < 5% | 月利5%で最大DD 5%以下は不可能に近い。損益プロファイルの異常性 | 中 |
| 6 | **取引期間（短い）** | バックテスト期間 < 6ヶ月 | 相場サイクル未網羅。1シーズンの運不要素だけでは検証不充分 | 中 |

**複合判定ロジック**:
- 上記 1～3 のいずれか1つでもヒット → **NO-GO**
- 上記 4～6 のうち2つ以上でヒット → **要確認**（デューデリジェンス要）
- すべて OK → **GO** 判定へ進む

#### 成績劣化シミュレーション（バックテスト≠リアル対応）

根拠: 9行目「スプレッド変動・スリッページ（+3～10pips）・リクオート・時刻ずれ」、46行目「リアル稼働では20~40%の成績悪化が典型」

| 項目 | バックテスト期待値 | リアル予想値（-30%）| 判定基準 |
|------|------------------|------------------|--------|
| 月利 | 5% | **3.5%** | リアル月利 < 3%なら NO-GO |
| 勝率 | 55% | **38.5%** | リアル勝率 < 40%なら警告 |
| PF | 1.8 | **1.26** | リアル PF < 1.2 なら NO-GO |
| Max DD | 18% | **23.4%** | リアル Max DD > 25% なら NO-GO |

→ **リアル稼働開始時のアラート基準**: バックテスト期待値の -35% を基準に月次監視

### 4.2 段階的GO/NO-GO判定フロー

```
[検証フロー]

Step 1: サンプルサイズ確認
├─ 取引数 < 500? → NO-GO（終了）
└─ >= 500 継続

Step 2: 危険シグナル複合判定
├─ PF >= 3.0 OR 勝率 >= 85% OR 最大連敗 <= 5? → NO-GO（終了）
└─ OK 継続

Step 3: バックテスト基準達成判定
├─ 月利 >= 3% AND 勝率 >= 50% AND Max DD <= 20%?
│  ├─ YES → Step 4 へ（GO基準検証）
│  └─ NO → 要確認（パラメータ見直し）

Step 4: GO基準チェック（上位25%目標）
├─ PF >= 1.8 AND Max DD <= 15% AND シャープ >= 1.0?
│  ├─ YES → **GO判定** → リアル稼働ロード
│  └─ NO → **要確認** → ウォークフォワード再検証

Step 5: ウォークフォワード分析（IS/OOS 70:30）
├─ OOS vs IS 乖離 <= -40%?
│  ├─ YES → **GO判定** → リアル稼働開始（月次監視）
│  └─ NO → **NO-GO** → ロジック見直し必須
```

#### パターン1：聖杯EA（バックテスト最適化過剰）

```
バックテスト結果:

- 取引数: 180回
- Max DD: 3%

      過剰最適化でフォワード崩壊80%超の可能性
```

#### パターン2：環境特異EA（パラメータドリフト）

```
- PF: 1.9

- Max DD: 12%
```

## 5. 統合サマリー：抽出結果の要約

### 5.1 EAコンセプト推奨案（MVP用）

**選定**: **レンジ型（ボリンジャーバンド+RSI逆張り）**

**理由**:
1. MaxDD 10～15% で 20% 制約を余裕でクリア
2. 高勝率（60%以上）で心理的安定性高い
3. 取引機会が多く MVP の 2000 回サンプル達成が 6～8 ヶ月で可能
4. 実装難度低・パラメータ少ない

### 5.2 MVP検証の最小スコープ

| 項目 | 仕様 |
|------|------|
| **通貨ペア** | USD/JPY（スプレッド0.2銭・流動性標準） |
| **インジケーター** | ボリバン（20,2σ）+ RSI（14,30/70）+ ATR（14） |
| **バックテスト期間** | 直近12ヶ月 |
| **Tick品質** | 粗検：Open prices only / 精細：Every tick based on real ticks |
| **スプレッド設定** | 0.3銭（乖離考慮） |
| **スリッページ設定** | 固定+5pips |
| **目標取引数** | 2,000回以上 |
| **検証手法** | ウォークフォワード分析（IS/OOS 70:30分割） |

### 5.3 V1スケールアップの必須要素

1. **複数通貨ペア対応** （3～5ペア）
2. **相場局面フィルター** （ADXベース・トレンド判別）
3. **動的SL/TP調整** （ボラティリティ対応）
4. **ウォークフォワード検証フレームワーク** （月次自動再検証）

### 5.4 GO/NO-GO判定の即断基準

**GO 確定条件**:
- 取引数 ≥ 2,000
- PF 1.5～2.0 （3.0未満）
- 勝率 50～70% （85%未満）
- Max DD ≤ 20% （実績 ≤ 15%）
- ウォークフォワード OOS vs IS 乖離 ≤ -40%

**NO-GO 判定条件（即終了）**:
- 取引数 < 500
- PF ≥ 3.0 OR 勝率 ≥ 85%
- Max DD > 25%
- ウォークフォワード乖離 > -40%

## 附録: キーワード定義

- **MaxDD（最大ドローダウン）**: ピークから谷までの最大損失。20%制約＝ピークから20%以上の損失が発生してはいけない
- **Profit Factor**: 総利益 ÷ 総損失。1.0は損益ゼロ。1.3以上で利益的
- **シャープレシオ**: 年間リターン ÷ リターンの標準偏差。リスク調整後のリターン指標
- **勝率**: 利益トレード数 ÷ 全トレード数
- **過剰最適化**: バックテスト期間内のノイズまで学習→他の相場環境では機能しない
- **ウォークフォワード分析**: 全期間を複数の in-sample / out-of-sample 区間に分割→各区間で最適化と検証を繰り返す
- **Tick Data Suite**: MT4バックテストの精度を上げるための高品質Tickデータ・テストツール
- **IS/OOS**: In-Sample（学習期間）/ Out-Of-Sample（検証期間）
